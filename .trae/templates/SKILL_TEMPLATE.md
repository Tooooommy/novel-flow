## 2. 命令规范

### 命令结构

```markdown
### {command-name}

**功能说明**
```

/nf-[skill-name] {command-name} --param1 <value> [--optional <value>]

````

**参数说明**:
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| param1 | string | 是 | - | 参数1说明 |
| optional | string | 否 | default | 可选参数 |

**执行流程**: step1 → step2 → step3

**示例**:
```bash
/nf-[skill-name] {command-name} --param1 "value"
````

---

### full 命令（必须）

所有 Skill 必须提供 `full` 一键执行命令：

```markdown
### full

**一键全量执行**
```

/nf-[skill-name] full --input <required-value>

```

**参数说明**:
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| input | string | 是 | - | 输入内容 |

**执行流程**: command1 → command2 → command3 → ...

**适用场景**: 快速执行完整流程，无需逐个调用子命令
```

### 命令执行顺序规范

| 命令类型 | 执行顺序   | 说明                     |
| -------- | ---------- | ------------------------ |
| `full`   | 第一位     | 一键执行，优先显示       |
| 主命令   | 2, 3, 4... | 按功能模块排列           |
| 原子命令 | 最后       | analyze, status, help 等 |

### 参数类型规范

| 类型    | 说明      | 示例                         |
| ------- | --------- | ---------------------------- |
| string  | 字符串    | `"hello"`                    |
| number  | 数字      | `42`                         |
| boolean | 布尔值    | `true` / `false`             |
| enum    | 枚举值    | `small` / `medium` / `large` |
| file    | 文件路径  | `./output.txt`               |
| json    | JSON 对象 | `{"key": "value"}`           |

### 参数值约束

| 约束             | 说明     | 示例                     |
| ---------------- | -------- | ------------------------ |
| `<value>`        | 必填参数 | `--topic <主题>`         |
| `[value]`        | 可选参数 | `--count [5]`            |
| `value1\|value2` | 枚举值   | `--type <ending\|twist>` |

## 5. 错误处理规范

### 错误码规范

```markdown
## 错误处理

| 错误码  | 等级  | 说明         | 处理方式             |
| ------- | ----- | ------------ | -------------------- |
| ERR-001 | ERROR | 主题为空     | 提示用户输入有效主题 |
| ERR-002 | WARN  | 概念数量不足 | 提示调整 count 参数  |
| ERR-003 | INFO  | 无效参数     | 显示帮助信息         |
```

### 错误等级

| 等级    | 说明     | 行为               |
| ------- | -------- | ------------------ |
| `FATAL` | 致命错误 | 终止执行，记录日志 |
| `ERROR` | 操作错误 | 回滚操作，返回错误 |
| `WARN`  | 警告     | 继续执行，提示警告 |
| `INFO`  | 信息     | 仅记录，不中断     |

## 8. 最佳实践

### Prompt 编写规范

```markdown
## Prompt 编写规范

1. **角色定义**: 明确 AI 扮演的角色
2. **任务描述**: 清晰说明要完成的任务
3. **输入格式**: 指定输入数据的格式
4. **输出格式**: 指定输出数据的格式
5. **约束条件**: 列出必须遵守的规则
6. **示例**: 提供 1-2 个正面示例
7. **去 AI 味**: 强调自然、真实的表达
```

### 去 AI 味指南

```markdown
### ❌ AI 味道特征（必须避免）

- 过度使用连接词（然而、因此、但是）
- 句式过于规整对仗
- 形容词堆砌
- 情感表达过于直白
- 缺乏生活细节

### ✅ 人类化写作特征

- 句子长短错落
- 口语化表达
- 动作描写优先
- 留白与暗示
- 情绪克制表达
```
