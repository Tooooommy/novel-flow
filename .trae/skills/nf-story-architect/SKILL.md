# 故事架构技能

## 使用场景

- 用户要生成小说大纲
- 用户要设计世界观设定
- 用户要创建人物设定
- 用户要生成分卷大纲

---

## 命令

### 生成完整大纲

> 根据 `novel.yaml` 项目元数据，生成完整大纲体系

```
/nf-story-architect
```

自动读取 `novel.yaml` 项目元数据文件，包含小说的题材、字数、分卷数量、每卷章节数等信息。

**执行流程**:

1. 读取 `novel.yaml` 项目元数据文件和用户选定的创意题材
2. 执行world流程构建完整世界观体系、地理设定、势力分布等
3. 执行characters流程设计主要人物设定等
4. 执行structure流程规划主线、副本、章节规划等
5. 执行outline流程生成完整主线大纲
6. 执行volume流程生成分卷大纲

## 流程

### world - 构建世界观

**执行方案**:

| 步骤 | 操作                   | 产出         |
| ---- | ---------------------- | ------------ |
| 1    | 确定题材类型和核心设定 | 题材定位     |
| 2    | 设计力量体系           | 力量等级文档 |
| 3    | 构建地理/空间设定      | 地图/场景    |
| 4    | 设计势力分布           | 势力关系图   |
| 5    | 制定世界规则           | 规则文档     |

**世界观构建标准**:

| 维度     | 必须包含           | 可选包含           | 禁止包含          |
| -------- | ------------------ | ------------------ | ----------------- |
| 力量体系 | 等级名称、升级条件 | 特殊能力、血脉     | 过于复杂（>20级） |
| 地理设定 | 主要区域、势力分布 | 气候、资源分布     | 细节过于琐碎      |
| 势力分布 | 主要势力、阵营对立 | 势力历史、恩怨     | 势力过多（>10个） |
| 世界规则 | 核心法则、限制设定 | 例外情况、稀有现象 | 规则冲突          |

**力量体系设计标准**:

| 力量类型 | 层级设计   | 升级方式       | 特色设计   |
| -------- | ---------- | -------------- | ---------- |
| 修仙流   | 7-9级      | 功法+资源+悟性 | 境界壁垒   |
| 武侠流   | 5-7级      | 秘籍+实战+内功 | 招式变化   |
| 异能流   | 3-5级+变异 | 天赋觉醒+开发  | 能力克制   |
| 魔法流   | 6-10级     | 元素+咒语+魔力 | 魔法代价   |
| 科技流   | 初级→高级  | 研发+资源+迭代 | 科技树分支 |

```markdown
【力量等级标准模板】

├─ 体质特殊: 天生异能或缺陷
└─ 外力加持: 丹药/法器/传承
```

### characters - 设计人物

**执行方案**:

| 步骤 | 操作                 | 产出     |
| ---- | -------------------- | -------- |
| 1    | 确定主角核心设定     | 主角定位 |
| 2    | 设计主角性格和成长线 | 成长弧   |
| 3    | 设计女主/重要配角    | 人物设定 |
| 4    | 设计反派/敌对势力    | 反派设定 |
| 5    | 确定人物关系网络     | 关系图谱 |

**人物设计标准**:

| 人物类型 | 核心属性           | 必写内容                 | 可选内容       |
| -------- | ------------------ | ------------------------ | -------------- |
| 主角     | 性格、目标、金手指 | 身份、性格、成长线、目标 | 外貌、背景故事 |
| 女主     | 性格、情感         | 身份、性格、与主角关系   | 外貌、身世     |
| 配角     | 定位、功能         | 身份、功能、特点         | 外貌、背景     |
| 反派     | 威胁、动机         | 身份、威胁、动机         | 外貌、背景     |

**主角设计标准**:

```markdown
【主角设计标准】

必要元素:
├─ 身份定位: 起点身份 → 终点身份
├─ 核心性格: 1个主性格 + 2-3个副性格
├─ 成长曲线: 起点 → 转折 → 中段 → 高潮 → 结局
├─ 金手指: 具体能力 + 成长性 + 代价
└─ 目标体系: 短期目标 / 中期目标 / 长期目标

可选元素:
├─ 外貌特征: 3-5个特征点
├─ 背景故事: 1-2个核心事件
├─ 心理弱点: 1个（用于制造冲突）
└─ 独特习惯: 1-2个（增加辨识度）

禁止元素:
├─ 完美人设（必须有弱点）
├─ 无代价能力（能力必须有约束）
└─ 目标模糊（必须有清晰目标）
```

**女主设计标准**:

```markdown
【女主设计标准】

必要元素:

├─ 核心性格: 与主角形成互补或对比
├─ 情感线: 发展过程 + 关键节点

├─ 误解: 制造冲突

├─ 青梅竹马型: 一直陪伴
```

| 工具型   | 推动情节 | 5-10%    | 随情节处理    |

**反派设计标准**:

```markdown

└─ 最终结局: 必须有交代

反派层级:
├─ 初级反派: 前30%章节
├─ 中级反派: 中期boss
└─ 终极反派: 最终boss

动机设计:
├─ 复仇型: 被主角或其势力伤害
├─ 利益型: 争夺资源/地位
├─ 信念型: 与主角理念冲突
├─ 命运型: 天生死敌
└─ 扭曲型: 心理扭曲/执念
```

### structure - 篇章结构

**执行方案**:

| 步骤 | 操作                 | 产出     |
| ---- | -------------------- | -------- |
| 1    | 确定总卷数和总章节数 | 结构框架 |
| 2    | 规划每卷的主题和目标 | 卷主题   |
| 3    | 设计每卷的起承转合   | 卷结构   |
| 4    | 规划章节间的衔接     | 章节关联 |
| 5    | 确定爽点/高潮分布    | 节奏规划 |

**篇章结构标准**:

| 结构类型 | 适用题材  | 特点           | 章节分布 |
| -------- | --------- | -------------- | -------- |
| 三段式   | 升级流    | 铺垫→高潮→爆发 | 1:2:1    |
| 递进式   | 悬疑/探案 | 层层剥茧       | 1:1:1    |
| 单元式   | 单元剧    | 独立故事串联   | 均匀分布 |
| 混合式   | 复杂剧情  | 多种结构混合   | 灵活     |

**章节数规划标准**:

| 总字数目标 | 每卷章节数 | 卷数 | 总章节数 |
| ---------- | ---------- | ---- | -------- |
| 30万字     | 100章      | 3卷  | 300章    |
| 50万字     | 100章      | 5卷  | 500章    |
| 100万字    | 100章      | 10卷 | 1000章   |

**三段式结构详解**:

```markdown
【三段式结构标准】

第一卷【建立期】(30-35%篇幅):
├─ 功能: 建立世界、塑造主角、引出势力
├─ 重点: 主角困境、金手指获得、第一个爽点
├─ 节奏: 慢→中→快
└─ 章节分布:
├─ 1-5章: 困境建立（慢节奏）
├─ 6-15章: 成长起步（中节奏）
├─ 16-25章: 小试牛刀（节奏加快）
└─ 26-30章: 第一个高潮（快节奏）

第二卷【成长期】(40-45%篇幅):
├─ 功能: 格局扩大、矛盾升级、势力扩张
├─ 重点: 实力提升、势力壮大、感情线发展
├─ 节奏: 铺垫→冲突→高潮→转折
└─ 章节分布:
├─ 31-40章: 新挑战引入
├─ 41-60章: 快速成长期
├─ 61-80章: 重大危机
└─ 81-90章: 重大转折

第三卷【高潮期】(25-30%篇幅):
├─ 功能: 最终对决、收束伏笔、完美结局
├─ 重点: 最终boss、情感圆满、所有伏笔回收
├─ 节奏: 连环高潮→最终对决→圆满收尾
└─ 章节分布:
├─ 91-100章: 最终boss出现
├─ 101-130章: 连环高潮
├─ 131-150章: 最终对决
└─ 151-160章: 圆满结局
```

**副本结构设计**:

| 副本类型 | 章节跨度 | 功能          | 爽点设计   |
| -------- | -------- | ------------- | ---------- |
| 历练副本 | 3-5章    | 获得资源/技能 | 突破、收获 |
| 比赛副本 | 5-10章   | 展示实力      | 夺冠、打脸 |
| 势力副本 | 10-20章  | 扩张势力      | 收服、壮大 |
| 探秘副本 | 5-10章   | 推进主线      | 揭露真相   |

| 6    | 生成每卷的章节大纲   | 章节大纲 |

**分卷结构设计标准**:

| 阶段             | 维度       | 要点                                                                                                   |
| ---------------- | ---------- | ------------------------------------------------------------------------------------------------------ |
| **S1: 卷数确定** | 字数预估法 | 总字数 = 卷数 × 每卷章数 × 每章字数；短篇(30-60万)/中篇(60-100万)/长篇(100-300万)                      |
|                  | 剧情阶段法 | 例: 觉醒→成长→小高潮→挫折→崛起→高潮 = 6卷                                                              |
|                  | 市场适应法 | 起点: 3000字/天，100万字上架；晋江: 2000字/天，VIP需30万字                                             |
| **S2: 每卷章数** | 建议范围   | 30-60章/卷，最优40-50章/卷                                                                             |
|                  | 分配原则   | 首卷25-35章(引入期)；中间卷40-50章(高潮积累)；终卷35-45章(收尾)                                        |
| **S3: 卷主题**   | 主题设计   | 每卷独立主题；贯穿情感线                                                                               |
|                  | 主题示例   | 卷1:觉醒/困境；卷2:成长/初试锋芒；卷3:崛起/小高潮；卷4:危机/挫折；卷5:觉醒/再崛起；卷6:最终对决/大高潮 |
| **S4: 章节规划** | 字数标准   | 单章2000-3000字                                                                                        |
|                  | 章内结构   | 开篇钩子→铺垫(300-500字)→发展(1000-1500字)→高潮/转折(500-800字)→结尾悬念(100-200字)                    |
|                  | 类型分布   | 推进型60%(主线)；铺垫型25%(世界观/情感)；高潮型15%(大场面/转折)                                        |

**分卷主题模板库**:

| 卷号 | 主题类型        | 核心内容                       | 爽点类型           | 章数建议 |
| ---- | --------------- | ------------------------------ | ------------------ | -------- |
| 1    | 觉醒/困境开局   | 主角出场、困境设定、金手指觉醒 | 身份反转、扮猪吃虎 | 25-35章  |
| 2    | 成长/初试锋芒   | 首次大战、势力认知、实力积累   | 小规模胜利、打脸   | 35-45章  |
| 3    | 崛起/小高潮     | 势力成型、地位提升、名声传播   | 越级挑战、势力扩张 | 40-50章  |
| 4    | 危机/挫折       | 强大敌人、失败、低谷期         | 绝境求生、隐藏实力 | 40-50章  |
| 5    | 觉醒/再崛起     | 领悟、突破、反转               | 复仇反击、实力认证 | 40-50章  |
| 6    | 最终对决/大高潮 | 终极boss、胜利、完美结局       | 终极对决、圆满收尾 | 35-45章  |

## 创作标准汇总

**大纲自检清单**:

| 检查项     | 标准                           | 不合格表现       |
| ---------- | ------------------------------ | ---------------- |
| 世界观完整 | 力量体系、地理、势力、规则都有 | 缺胳膊少腿       |
| 人物立体   | 主角有缺点、反派有动机         | 脸谱化           |
| 主线清晰   | 一句话能说清                   | 支线过多干扰主线 |
| 节奏合理   | 有起有伏、高潮分布均匀         | 平铺直叙或跳跃   |
| 伏笔可控   | 埋的坑都能填                   | 坑太多忘记填     |
| 爽点充足   | 目标字数/1000 ≥ 爽点数         | 憋屈、水字数     |

**大纲质量评估**:

| 维度   | 优秀         | 合格     | 不合格     |
| ------ | ------------ | -------- | ---------- |
| 世界观 | 独特且自洽   | 基本完整 | 有明显漏洞 |
| 人物   | 鲜活有记忆点 | 基本立体 | 脸谱化     |
| 情节   | 跌宕起伏     | 基本流畅 | 平淡或混乱 |
| 逻辑   | 环环相扣     | 基本合理 | 漏洞百出   |
| 爽点   | 密集且多样   | 有爽点   | 憋屈       |

## 错误处理

| 错误码 | 说明     | 处理方式                     |
| ------ | -------- | ---------------------------- |
| SA-001 | 题材为空 | 使用默认题材（玄幻废柴逆袭） |
| SA-002 | 卷数无效 | 默认3卷                      |
| SA-003 | 章节超限 | 按比例自动调整               |
| SA-004 | 人物冲突 | 优先保证主角，次要人物合并   |
| SA-005 | 设定矛盾 | 以主线为准，自动修复         |
| SA-006 | 伏笔遗漏 | 自动补充回收节点             |
| VM-001 | 大纲为空 | 使用默认结构(3卷标准模板)    |
| VM-002 | 卷数无效 | 默认3卷                      |
| VM-003 | 章节超额 | 自动调整或提示拆分           |
| VM-004 | 主题重复 | 提示重新设计卷主题           |

**错误恢复流程**:

```
1. 检测错误类型
     │
     ├──→ SA-001 → 使用默认玄幻题材模板
     │
     ├──→ SA-002/003 → 自动计算合理分配
     │
     ├──→ SA-004 → 角色合并或删除
     │
     ├──→ SA-005 → 以主线逻辑为准修复
     │
     └──→ SA-006 → 补充伏笔回收计划
```
