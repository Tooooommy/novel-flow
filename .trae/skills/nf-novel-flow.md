# Skill: nf-novel-flow

## 描述

Novelflow 小说创作流程主控技能，6个核心命令覆盖小说创作全流程，支持分卷大纲、分卷创作，卷与卷之间自动衔接，支持章节/行级别精细化操作。

## 调用方式

```
/nf <command> [options]
```

---

## 核心命令

### idea

**创意探索** - AI生成多个创意方案供选择

```
/nf idea [--count <数量>] [--genre <题材方向>]
```

**参数说明**:
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| count | number | 否 | 5 | 生成创意数量 |
| genre | string | 否 | - | 题材方向 |

**智能行为**:

- 分析当前热门网文趋势
- 结合用户偏好生成创意
- 提供多个差异化的创意方案

**输出示例**:

```

│ 创意方案                                        │
├─────────────────────────────────────────────────┤
│ [1] 都市修仙：废物逆袭，校花倒贴                │
│     标签：都市｜修仙｜热血｜爽文                 │
│     亮点：身份反差+系统流+暧昧流                │
│                                                 │
│ [2] 异世穿越：成为史上最弱召唤师               │
│     标签：奇幻｜穿越｜召唤流｜搞笑             │
│     亮点：反套路+宠物成长+轻松向               │
│                                                 │
│ [3] 都市神医：开局一座山，全是退休大佬         │
│     标签：都市｜神医｜经营｜装逼流             │
│     亮点：身份隐藏+人脉经营+医术+鉴宝          │
```

### init

**小说立项** - 创建新小说项目

```
/nf init --name <名称> --genre <题材> [--target <字数>]
```

**参数说明**:
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| name | string | 是 | - | 小说名称 |
| genre | string | 是 | - | 题材类型 |
| target | number | 否 | 100000 | 目标字数 |

**智能行为**:

- 自动创建项目目录结构
- 自动生成项目配置文件
- 自动调用市场分析（可选）

---

### outline

**大纲生成** - 同时生成总大纲和分卷大纲

```
/nf outline [--chapters <数量>] [--volumes <卷数>]
```

**参数说明**:
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| chapters | number | 否 | 30 | 每卷章节数量 |
| volumes | number | 否 | 3 | 分卷数量 |

**智能行为**:

- 读取项目配置自动获取题材类型
- 自动生成总大纲结构
- 自动拆分生成分卷大纲
- 自动规划各卷字数分配

**输出结构**:

```
novels/{project}/
├── outline.md          # 总大纲
├── volume-1.md         # 第一卷大纲
├── volume-2.md         # 第二卷大纲
└── volume-3.md         # 第三卷大纲
```

### write

**正文创作** - 按分卷创作，自动审查循环

```
/nf write [--volume <n>] [--chapter <n>] [--lines <start-end>]
```

**参数说明**:
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| volume | number | 否 | 当前卷 | 分卷号 |
| chapter | number | 否 | 自动 | 章节号 |
| lines | string | 否 | 全部 | 行范围 |

**智能行为**:

- 无参数时：自动找到当前卷最后一个未完成章节
- 自动加载对应分卷大纲
- 自动衔接上一卷结尾内容（承上启下）
- 自动应用已学习的文风设置

**自动审查循环**:

```
写作 ──→ 审查 ──→ 有问题? ──→ 是 ──→ 优化 ──→ 审查
                │                        │
                └──────── 否 ─────────────┘
                              ↓
                          下一章
                          ↓
                     (换卷时自动衔接优化)
```

- 每次写作后自动执行审查
- 有问题自动优化后重新审查
- 没问题自动继续下一章
- 循环直至当前卷完成
- 换卷时自动执行衔接优化（情节/文风）

---

### review

**内容审批** - 自动扫描全文质量问题

```
/nf review [--chapter <n>] [--level <standard|strict>]
```

**参数说明**:
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| chapter | number | 否 | 全部 | 章节号 |
| level | string | 否 | standard | 审查级别 |

**智能行为**:

- 无参数时：自动扫描全文章节问题
- 自动分类问题类型（逻辑/风格/合规）
- 自动给出修复建议

## 智能命令

### auto

**一键自动创作** - 智能检测+执行完整流程

```
/nf auto
```

**智能行为**:

1. 检测项目当前状态
2. 自动识别待处理章节
3. 执行 write → review → optimize
4. 循环处理直至完成
5. 生成创作报告

---

### detect

**状态检测** - 分析项目并给出待办列表

```
/nf detect [--scope <pending|issues|all>]
```

**参数说明**:
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| scope | string | 否 | all | 检测范围 |

**智能行为**:

- `pending`: 列出未完成的章节
- `issues`: 列出需要修复的问题
- `all`: 完整项目状态报告

**输出示例**:

```
┌─────────────────────────────────────────┐
│ 项目状态报告                             │
├─────────────────────────────────────────┤
│ 📖 进度: 12/30 章 (40%)                 │
│ 📝 待完成: 第13-30章                     │
│ ⚠️ 问题: 3处待修复                       │
│   - 第5章 逻辑矛盾                       │
│   - 第7章 节奏过慢                       │
│   - 第11章 敏感词                        │
│ ✅ 已完成: 第1-12章                      │
└─────────────────────────────────────────┘
```

## 精细化命令

### optimize

**优化指定内容** - 针对章节、段落或卷间衔接

```
/nf optimize [--chapter <n>] [--volume <n>] [--type <类型>]
```

**参数说明**:
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| chapter | number | 否 | - | 章节号 |
| volume | number | 否 | - | 分卷号 |
| type | string | 否 | all | 优化类型 |

**type 类型说明**:
| 类型 | 说明 |
|------|------|
| all | 全部优化 |
| style | 文风优化（去AI味道） |
| plot | 情节优化（逻辑/节奏） |
| connection | 卷间衔接优化 |
| merge | 衔接+统一文风 |

**智能行为**:

- 自动识别需要优化的内容类型
- 自动应用去AI味道处理
- 自动调整节奏和风格
- 自动优化卷与卷之间的衔接（情节/文风）

---

**参数说明**:
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| issue | string | 是 | - | 问题ID |

**智能行为**:

- 自动定位问题位置
- 自动分析问题原因
- 自动生成修复方案

## 创作流程

```
idea → init → outline(含分卷) → write(按卷自动衔接) → optimize → review → publish
```

### 模式一：灵感触发模式（不知道写什么）

```
/nf idea --genre 玄幻 --count 3     ← AI提供创意
/nf init --name "xxx" --genre 玄幻  ← 选择后立项
/nf outline --volumes 3              ← 生成3卷大纲
/nf write                           ← 自动按卷创作（自动衔接）
/nf optimize                         ← 优化全文
/nf review                          ← 审批全文
/nf publish --platform 起点         ← 发布
```

### 模式二：快速自动模式

```
/nf idea
/nf init --name "xxx" --genre 玄幻
/nf outline --volumes 5
/nf auto                            ← 自动完成：写→优化→审批
/nf publish --platform 起点
```

### 模式三：手动精细化模式

```
/nf idea
/nf init --name "xxx" --genre 玄幻
/nf outline --volumes 3 --chapters 20
/nf write --volume 1 --chapter 1     ← 指定第1卷第1章
/nf write --volume 2                 ← 继续第2卷（自动衔接）
/nf optimize --chapter 10 --lines 50-100 ← 精细优化
/nf review
/nf publish --platform 晋江
```

### 模式三：目标明确模式

```
/nf init --name "我的小说" --genre 玄幻
/nf outline --volumes 3
/nf auto
```

## 示例

### 1. 创建新小说项目

```
/nf init --name "凡人修仙传" --genre 仙侠 --target 300000
```

### 2. 生成大纲

```
/nf outline --chapters 50
```

### 3. 手动创作第一章

```
/nf write --chapter 1
```

### 4. 优化第3章10-50行

```
/nf optimize --chapter 3 --lines 10-50
```

### 5. 审查全文章节

```
/nf review
```

### 6. 修复第5章的逻辑问题

```
/nf detect --scope issues
/nf fix --issue LOGIC-005
```

### 7. 发布到起点中文网

```
/nf publish --platform qidian
```
