# Skill: nf-style-learner

## 描述

风格学习技能，允许用户提供文风样本，让系统学习并模仿其写作风格、节奏和用词偏好。

## 调用方式

```
/nf-style-learner <command> [options]
```

## 命令

### learn

学习文本风格

```
/nf-style-learner learn --text <sample-text> [--name <style-name>]
```

**参数说明**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| text | string | 是 | 风格样本文本 |
| name | string | 否 | 风格名称 |

**输出**: 风格特征报告

---

### analyze

分析已保存的风格

```
/nf-style-learner analyze --style <style-id>
```

---

### apply

应用风格到内容

```
/nf-style-learner apply --content <content> --style <style-id>
```

---

### full ⭐NEW

**一键完整风格学习流程**

```
/nf-style-learner full --text <sample-text> --content <content-to-transform>
```

**执行流程**: learn → analyze → apply

## 风格特征维度

### 1. 句式特征

| 维度 | 说明 | 量化指标 |
|------|------|---------|
| 平均句长 | 平均每句字数 | <15短/15-25中/>25长 |
| 短句比例 | 短句占总句数比例 | <20%低/20-40%中/>40%高 |
| 句式变化 | 长短句交替模式 | 单一/略有变化/丰富 |
| 段落长度 | 平均段落句数 | <3短/3-6中/>6长 |

### 2. 用词特征

| 维度 | 说明 | 量化指标 |
|------|------|---------|
| 词汇复杂度 | 生僻词/高级词使用频率 | 低/中/高 |
| 形容词密度 | 每百字形容词数 | <5低/5-10中/>10高 |
| 动词多样性 | 独特动词使用比例 | <30%低/30-50%中/>50%高 |
| 口语化程度 | 口语词汇使用比例 | <10%正式/10-30%一般/>30%口语 |

### 3. 叙事特征

| 维度 | 说明 | 量化指标 |
|------|------|---------|
| 视角 | 人称使用 | 第一/第二/第三 |
| 展示vs陈述 | Show/Tell比例 | <30%展示/30-50%混合/>50%展示 |
| 对话比例 | 对话占总内容比例 | <20%少/20-40%适中/>40%多 |
| 内心描写 | 心理活动比例 | <10%少/10-25%适中/>25%多 |

### 4. 情感特征

| 维度 | 说明 | 量化指标 |
|------|------|---------|
| 情感强度 | 强烈情感词频率 | 低/中/高 |
| 情感表达方式 | 直接/间接 | 直接陈述/动作暗示 |
| 幽默元素 | 幽默/讽刺使用 | 无/偶尔/经常 |

## 风格模板

### 1. 简洁干练型

```
特征:
  平均句长: 15-20字
  短句比例: >50%
  形容词密度: <5
  对话比例: >40%
  展示vs陈述: >60%展示

示例:
  他推开门。屋里没人。桌上放着一封信。
  他拆开。看完。脸色变了。
```

### 2. 细腻绵长型

```
特征:
  平均句长: >30字
  短句比例: <20%
  形容词密度: >10
  对话比例: <20%
  展示vs陈述: >70%展示

示例:
  他缓缓地推开了那扇略显陈旧的木门,门轴发出轻微的吱呀声,
  像是在诉说着这座老宅所经历的漫长岁月与无数故事。
```

### 3. 悬疑紧张型

```
特征:
  平均句长: 15-25字
  短句比例: 40-60%
  悬念句比例: >30%
  情感强度: 中-高
  短段落多: <3句/段

示例:
  脚步声停了。
  他屏住呼吸，心跳如鼓。
  黑暗中，似乎有什么东西在靠近。
```

### 4. 轻松幽默型

```
特征:
  对话比例: >50%
  短句比例: >50%
  幽默元素: 经常
  口语化程度: >40%
  感叹句比例: >15%

示例:
  "我说老王啊，你这车技可真是一绝！"他扶着车门，
  脸色发白，"下次能不能别当碰碰车开？"
```

## 学习流程详解

### Step 1: 文本采集

**要求**:
- 最少提供 1000 字样本
- 最好来自同一作品的多章节
- 避免混用不同风格的内容

**采集指南**:
- 提供最能代表您风格的段落
- 包括对话、描写、叙述等不同类型
- 如有特别擅长的场景类型，优先提供

### Step 2: 特征提取

**提取内容**:
1. 句式统计
   - 句子长度分布
   - 句型种类（陈述/疑问/感叹/祈使）
   - 段落结构

2. 词汇分析
   - 高频词汇表
   - 特色用词
   - 禁用词（作者避免使用的词）

3. 修辞识别
   - 常用修辞手法
   - 比喻/拟人频率
   - 对仗/排比使用

4. 风格标签
   - 生成5-10个风格标签
   - 如：简洁控/对话控/慢热型等

### Step 3: 模板生成

**输出格式**:
```yaml
style_template:
  id: "style-[uuid]"
  name: "自定义风格名"
  source: "样本来源"

  features:
    sentence:
      avg_length: 18
      short_ratio: 0.55
      variation: "high"

    vocabulary:
      complexity: "medium"
      adj_density: 6
      verb_diversity: 0.45
      colloquial: 0.35

    narrative:
      perspective: "third"
      show_ratio: 0.65
      dialogue_ratio: 0.40
      inner_ratio: 0.15

  writing_rules:
    - "优先使用短句"
    - "对话直接引用，不加提示语"
    - "减少形容词使用"
    - "动作描写用具体动词"

  avoid_list:
    - "非常"
    - "极其"
    - "完美无缺"
    - "喃喃自语"

  prompt_extension: |
    你的写作风格要求：
    1. 简洁有力，优先短句
    2. 对话自然，避免"说道"等提示语
    3. 用动作展示情绪，不直接陈述
    4. 形容词适度，不堆砌
```

### Step 4: 风格应用

**应用方式**:
1. 在内容生成时注入 prompt_extension
2. 使用 writing_rules 过滤输出
3. 避免使用 avoid_list 中的词汇

## 限制与注意事项

1. **样本质量**: 样本越能代表您的真实风格，效果越好
2. **风格迁移**: AI风格迁移只能接近，无法100%复制
3. **适用范围**: 对话多的内容风格迁移效果更明显
4. **更新迭代**: 可以多次学习同一风格进行强化

---

## 错误处理

| 错误码 | 说明 | 处理方式 |
|--------|------|---------|
| SL-001 | 样本字数不足 | 提示用户至少提供1000字 |
| SL-002 | 风格不存在 | 提示用户先创建风格 |
| SL-003 | 内容为空 | 提示用户提供要转换的内容 |
| SL-004 | 风格冲突 | 提示用户选择其他风格 |
