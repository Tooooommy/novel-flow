# 逻辑审查技能

## 使用场景

- 用户要审查章节逻辑
- 用户要检查时间线
- 用户要验证伏笔呼应
- 用户要检查设定一致性
- 用户要进行整体逻辑审查

---

## 命令

### full - 逻辑审查

```
/nf-logic-inspector full [--chapter <章节号>] [--volume <卷号>]
```

| 参数    | 类型   | 必填 | 默认值 | 说明   |
| ------- | ------ | ---- | ------ | ------ |
| chapter | number | 否   | 当前   | 章节号 |
| volume  | number | 否   | 当前   | 卷号   |

**执行流程**:

1. 调用consistency技能检查连贯性：
   - 检查事件因果关系是否合理
   - 检查人物行为是否符合性格
   - 检查能力表现是否前后一致
   - 检查信息传递是否合理
   - 检查物理逻辑是否符合基本物理
2. 调用timeline技能检查时间线：
   - 检查时间线是否连续
   - 检查时间线是否符合逻辑
3. 调用plotholes技能检查伏笔呼应：
   - 检查伏笔是否与事件因果关系一致
   - 检查伏笔是否与人物行为一致
   - 检查伏笔是否与能力表现一致
4. 输出逻辑审查报告

**输出**:

逻辑审查报告包含以下内容：

1. 是否通过审查
2. 问题列表
3. 警告列表
4. 时间线检查结果: 是否通过 + 时间线断裂位置
5. 伏笔检查结果: 是否通过 + 埋下的伏笔 + 未回收的伏笔列表
6. 修复建议

## 连贯性审查

### consistency - 连贯性

```
/nf-logic-inspector consistency [--chapter <章节号>] [--volume <卷号>]
```

**审查维度**:

| 维度     | 说明         | 检测方法      |
| -------- | ------------ | ------------- |
| 情节逻辑 | 事件因果关系 | 原因→结果检查 |
| 人物行为 | 行为符合性格 | 人设对照      |
| 能力表现 | 实力前后一致 | 战力对照      |
| 信息逻辑 | 信息传递合理 | 信息源分析    |
| 物理逻辑 | 符合基本物理 | 常识检查      |

**逻辑问题分类**:

| 问题类型 | 描述             | 示例                   | 严重程度 |
| -------- | ---------------- | ---------------------- | -------- |
| 因果矛盾 | 原因无法导出结果 | "因为没吃饭，所以饱了" | 严重     |
| 行为矛盾 | 行为与性格不符   | 懦弱主角突然勇敢       | 中等     |
| 能力矛盾 | 前后实力不符     | 前章被秒，后章秒杀     | 严重     |
| 信息矛盾 | 信息来源不明     | 无人告知却知道         | 中等     |
| 物理矛盾 | 违反物理规律     | 突然出现/消失          | 严重     |

**因果逻辑审查标准**:

| 检查项   | 标准               | 不合格表现 |
| -------- | ------------------ | ---------- |
| 事件因果 | 每个事件有明确原因 | 无因之果   |
| 行动动机 | 每个行为有合理动机 | 行为突兀   |
| 决策逻辑 | 决定符合已知信息   | 信息不一致 |
| 意外处理 | 意外事件有合理处理 | 强行化解   |

**人物行为审查标准**:

| 检查项   | 标准             | 不合格表现 |
| -------- | ---------------- | ---------- |
| 性格一致 | 行为符合设定性格 | 性格突变   |
| 能力一致 | 表现符合已知能力 | 突然增强   |
| 记忆一致 | 记得应该记得的   | 金鱼记忆   |
| 关系一致 | 态度符合关系现状 | 关系突变   |

**战力体系审查标准**:

```markdown
【战力一致性检查】

1. 相对实力对比
   - A > B > C
   - 若A胜B，则A必胜C
   - 除非有特殊条件

2. 绝对实力表现
   - 筑基期不可能战胜金丹期
   - 除非有特殊手段

3. 成长曲线
   - 实力增长需要时间和努力
   - 不能突兀增强

4. 克制关系
   - 元素相克需要提前铺垫
   - 不能临时添加
```

**检测流程**:

```
1. 提取本章所有事件
     │
     ├──→ 事件A: [内容]
     ├──→ 事件B: [内容]
     └──→ 事件C: [内容]

2. 检查每个事件的因果链
     │
     ├──→ 事件A原因?
     │     │
     │     ├──→ 有合理原因 → ✅
     │     └──→ 无原因/不合理 → ❌
     │
     ├──→ 事件B原因?
     │     │
     └──→ ...

3. 汇总问题
```

**问题描述模板**:

```markdown
【逻辑问题描述】

1. 问题位置: 第X段
   问题类型: [类型]
   原文: [相关原文]
   问题: [具体描述]
   原因: [为什么有问题]
   建议: [如何修复]
```

## 时间线审查

### timeline - 时间线

```
/nf-logic-inspector timeline [--chapter <章节号>] [--volume <卷号>]
```

**审查维度**:

| 维度     | 说明           | 检测方法   |
| -------- | -------------- | ---------- |
| 时间顺序 | 事件按时间排列 | 标记时间点 |
| 时间跳跃 | 跳跃有说明     | 检查过渡   |
| 季节变化 | 随时间合理变化 | 对照日期   |
| 年龄变化 | 角色年龄合理   | 记录年龄   |
| 节日假日 | 与设定一致     | 检查日期   |

**时间线问题分类**:

| 问题类型   | 描述               | 示例         | 严重程度 |
| ---------- | ------------------ | ------------ | -------- |
| 时间倒流   | 后面的事件早于前面 | B事件在A之前 | 严重     |
| 时间矛盾   | 同时发生的事件冲突 | 同一角色两处 | 严重     |
| 跳跃无说明 | 时间跳跃无过渡     | 直接多年后   | 中等     |
| 季节错误   | 与设定季节不符     | 夏天飘雪     | 中等     |

**时间线一致性标准**:

```markdown
【时间线标准】

1. 绝对时间线
   - 使用明确时间标记
   - 日-月-年 或 章数对应

2. 相对时间线
   - 事件顺序必须合理
   - 不能出现悖论

3. 季节设定
   - 与地理/气候一致
   - 跨度大时标注季节变化

4. 角色年龄
   - 记录关键时间点
   - 确保年龄合理增长
```

**时间标记模板**:

```markdown
【时间线标记】

| 章节   | 时间点   | 事件     | 备注   |
| ------ | -------- | -------- | ------ |
| 第1章  | 三月初一 | 主角入门 |        |
| 第5章  | 三月初五 | 宗门大比 | 4天后  |
| 第10章 | 三月十五 | 遗迹探索 | 10天后 |
```

**时间跳跃处理标准**:

| 跳跃类型       | 处理方式 | 示例            |
| -------------- | -------- | --------------- |
| 短跳跃(1-7天)  | 简略交代 | "数日后"        |
| 中跳跃(7-30天) | 说明原因 | "闭关一月后"    |
| 长跳跃(30天+)  | 详细说明 | "三年后，他..." |

## 伏笔审查

### plotholes - 伏笔

```
/nf-logic-inspector plotholes [--chapter <章节号>] [--volume <卷号>]
```

**伏笔管理**:

| 伏笔类型   | 说明           | 回收时机   |
| ---------- | -------------- | ---------- |
| 情节伏笔   | 关键剧情的前置 | 10-20章内  |
| 人物伏笔   | 人物身份的秘密 | 30-50章内  |
| 世界观伏笔 | 世界设定的揭示 | 50-100章内 |
| 情感伏笔   | 感情的铺垫     | 情感爆发前 |

**伏笔审查标准**:

| 检查项   | 标准          | 不合格表现 |
| -------- | ------------- | ---------- |
| 埋设清晰 | 有明显提示    | 埋设太隐晦 |
| 回收及时 | 在承诺范围内  | 忘记回收   |
| 回收合理 | 符合之前埋设  | 凭空出现   |
| 坑数控制 | 同时不超过5个 | 坑太多     |

**伏笔管理标准**:

```markdown
【伏笔管理标准】

1. 埋设标准
   - 伏笔必须有明确目的
   - 埋设时要有足够提示
   - 不能埋无法回收的伏笔

2. 回收标准
   - 短伏笔(5-10章)
   - 中伏笔(10-30章)
   - 长伏笔(30章以上)

3. 坑数控制
   - 同时进行中的坑 ≤ 5
   - 已回收的坑及时标记
   - 未回收的坑定期检查

4. 回收方式
   - 自然回收(情节推进)
   - 强制回收(节点必须)
   - 转换(转化为其他伏笔)
```

**伏笔追踪表**:

```markdown
【伏笔追踪表】

| 伏笔ID | 埋设章节 | 内容         | 预期回收 | 实际回收 | 状态   |
| ------ | -------- | ------------ | -------- | -------- | ------ |
| P001   | 第3章    | 神秘老者身份 | 第30章   | -        | 进行中 |
| P002   | 第5章    | 主角身世之谜 | 第50章   | -        | 进行中 |
| P003   | 第10章   | 宗门阴谋     | 第25章   | 第25章   | 已回收 |
```

**伏笔回收检查**:

| 检查项   | 标准             | 问题      |
| -------- | ---------------- | --------- |
| 回收逻辑 | 必须符合之前埋设 | 临时编造  |
| 回收时机 | 在合理时间范围内 | 太晚/太早 |
| 回收方式 | 自然合理         | 突兀      |
| 回收完整 | 全部回收         | 部分遗漏  |

## 设定一致性审查

**设定类型**:

| 设定类型   | 说明                 | 审查要点 |
| ---------- | -------------------- | -------- |
| 世界观设定 | 力量体系、地理、规则 | 是否冲突 |
| 人物设定   | 性格、能力、背景     | 是否违背 |
| 势力设定   | 关系、实力、规矩     | 是否矛盾 |
| 道具设定   | 能力、限制、归属     | 是否混乱 |

**设定一致性检查表**:

```markdown
【设定一致性检查】

1. 力量体系
   ├─ 等级设定是否一致
   ├─ 升级条件是否冲突
   └─ 战力对比是否合理

2. 地理设定
   ├─ 距离是否一致
   ├─ 位置是否矛盾
   └─ 势力范围是否重叠

3. 人物设定
   ├─ 性格是否一致
   ├─ 能力是否矛盾
   └─ 关系是否冲突

4. 道具设定
   ├─ 能力是否一致
   ├─ 归属是否明确
   └─ 数量是否混乱
```

## 错误处理

| 错误码 | 说明         | 处理方式       |
| ------ | ------------ | -------------- |
| LI-001 | 章节不存在   | 返回空结果     |
| LI-002 | 文本为空     | 跳过审查       |
| LI-003 | 信息不足     | 标记需要补充   |
| LI-004 | 矛盾无法解决 | 标记需人工判断 |

**错误恢复流程**:

```
1. 检测错误类型
     │
     ├──→ LI-001 → 返回"章节不存在"
     │
     ├──→ LI-002 → 返回"文本为空"
     │
     ├──→ LI-003 → 标记"信息不足"
     │
     └──→ LI-004 → 标记"需人工判断"
```
