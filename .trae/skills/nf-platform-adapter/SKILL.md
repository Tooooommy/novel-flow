## 命令

### 平台适配

> 适配小说到特定平台, 生成符合平台要求的元数据, 制定发布计划。

```
/nf-platform-adapter
```

| 参数     | 类型   | 必填 | 默认值 | 说明     |
| -------- | ------ | ---- | ------ | -------- |
| platform | string | 是   | -      | 目标平台 |

**执行流程**:

1. 调用metadata命令生成元数据：根据平台要求生成书名、简介、标签等元数据。
2. 调用strategy命令制定发布计划：根据平台要求制定发布计划，包括发布时间、发布频率、发布渠道等。

## 流程

### metadata - 元数据

**平台元数据规范**:

| 平台        | 书名长度 | 简介长度  | 标签数量 | 标签要求       |
| ----------- | -------- | --------- | -------- | -------------- |
| qidian      | 4-20字   | 100-500字 | 3-5个    | 精准、热门组合 |
| qidian-free | 4-20字   | 50-200字  | 3-5个    | 吸引眼球       |
| zongheng    | 4-30字   | 100-300字 | 3-8个    | 可含英文标签   |
| 17k         | 4-20字   | 50-300字  | 3-10个   | 宽松           |
| jinjiang    | 4-20字   | 100-300字 | 1-3个    | 精准、核心标签 |
| fanqie      | 4-20字   | 100-300字 | 1-3个    | 精准、核心标签 |

**书名生成标准**:

| 书名类型 | 结构特点      | 示例                         |
| -------- | ------------- | ---------------------------- |
| 直白型   | 题材+核心元素 | 《都市修仙录》《异界召唤师》 |
| 悬念型   | 引发好奇      | 《他其实很强》《消失的功法》 |
| 反差型   | 形成反差吸引  | 《废物逆袭》《我真不是高手》 |
| 组合型   | 多元素组合    | 《都市+修仙+奶爸》           |
| 热点型   | 蹭热点词      | 《开局XXX》《穿越XXX》       |

**简介生成标准**:

| 类别         | 要点                                                                                       |
| ------------ | ------------------------------------------------------------------------------------------ |
| **必备要素** | 主角身份(谁)、核心冲突(面临什么问题)、金手指/特色(有什么特别之处)、悬念/看点(为什么值得看) |
| **结构公式** | 身份(1-2句) + 困境(2-3句) + 转折(1-2句) + 看点(1-2句)                                      |

| 平台 | 风格                      |
| ---- | ------------------------- |
| 起点 | 偏商业化，强调爽点和看点  |
| 纵横 | 偏文学性，强调世界观      |
| 17K  | 偏小白文，直白易懂        |
| 晋江 | 偏文艺，强调情感和文笔    |
| 番茄 | 偏快节奏，强调前3秒吸引力 |

**标签系统**:

| 标签类型 | 说明          | 示例                     |
| -------- | ------------- | ------------------------ |
| 题材标签 | 小说所属题材  | 都市、玄幻、仙侠、奇幻   |
| 元素标签 | 核心元素      | 系统流、穿越、重生、星际 |
| 风格标签 | 文风/内容风格 | 热血、轻松、治愈、虐心   |
| 受众标签 | 目标读者群体  | 男频、女频、同人         |
| 时长标签 | 故事时间线    | 短篇、中篇、长篇，超长篇 |

### strategy - 发布策略

**发布周期标准**:

| 平台        | 建议日更      | 全勤要求   | 爆发策略           |
| ----------- | ------------- | ---------- | ------------------ |
| qidian      | 3000字(2更)   | 月更9万字  | 周末多更，冲榜     |
| qidian-free | 5000字(2-3更) | 月更15万字 | 日更5000+冲击全勤  |
| zongheng    | 3000字(2更)   | 月更9万字  | 编辑推荐前囤稿     |
| 17k         | 3000字(2更)   | 月更9万字  | 买断冲量           |
| jinjiang    | 2000字(1-2更) | 月更6万字  | 细水长流，保持更新 |

**冲榜时间表**:

| 榜单类型 | 投票/爆发时间       | 说明               |
| -------- | ------------------- | ------------------ |
| 周票榜   | 周一00:00-周日24:00 | 周一投效果最好     |
| 月票榜   | 月初-月末           | 月底最后3天冲刺    |
| 推荐票榜 | 每日0点更新         | 每天都可投         |
| 打赏榜   | 无固定时间          | 重要章节后效果更好 |
| 新书榜   | 签约后30天内        | 前14天关键期       |

**发布策略标准**:

| 阶段   | 时间        | 字数      | 策略重点                 |
| ------ | ----------- | --------- | ------------------------ |
| 公众期 | 签约前      | 日更3000+ | 积累读者，冲击新书榜     |
| 会员期 | 上架后第1月 | 日更3000+ | 保持月票排名，争全勤     |
| 成长期 | 上架后2-3月 | 日更3000+ | 冲击金票/分类榜          |
| 稳定期 | 4月+        | 日更3000+ | 保持稳定读者群，完本准备 |
| 完本期 | 完本前1-2月 | 放缓节奏  | 完本推荐，完本活动       |

## 错误处理

| 错误码 | 说明         | 处理方式                 |
| ------ | ------------ | ------------------------ |
| PA-001 | 平台不支持   | 使用通用格式，提示不支持 |
| PA-002 | 内容为空     | 返回空结果，提示输入内容 |
| PA-003 | 敏感词过多   | 分级处理，生成修改建议   |
| PA-004 | 格式转换失败 | 保留原文，提示错误原因   |
| PA-005 | 元数据过长   | 自动截取并提示超长部分   |

**错误恢复流程**:

```
PA-001:
  - 返回通用格式版本
  - 提示该平台的specific规范

PA-002:
  - 返回格式模板
  - 提示必填字段

PA-003:
  - 列出所有敏感词及位置
  - 提供修改建议
  - 给出修改后的预览

PA-004:
  - 保留原文
  - 记录转换失败的具体位置
  - 建议手动检查

PA-005:
  - 自动截取合规长度
  - 提示截断位置和原因
  - 建议精简方案
```
