# 平台适配技能

## 使用场景

- 用户要将小说发布到特定平台
- 用户要转换格式
- 用户要检查敏感词
- 用户要生成封面文案

---

## 命令

### full - 平台适配

```
/nf-platform-adapter full --platform <平台> [--chapter <章节号>] [--volume <卷号>]
```

| 参数     | 类型   | 必填 | 默认值 | 说明     |
| -------- | ------ | ---- | ------ | -------- |
| platform | string | 是   | -      | 目标平台 |
| chapter  | number | 否   | 当前   | 目标章   |
| volume   | number | 否   | 当前   | 目标卷   |

**执行流程**:

1. 调用format命令转换格式：按平台规范转换格式，包括章节标题、段落格式、标点规范、特殊要求等。
2. 调用sensitive命令检查敏感词：按平台敏感词规范检查，确保内容符合平台要求。
3. 调用metadata命令生成元数据：根据平台要求生成书名、简介、标签等元数据。
4. 调用strategy命令制定发布策略：根据平台要求制定发布计划，包括发布时间、发布频率、发布渠道等。

## 内部命令

### format - 格式转换

```
/nf-platform-adapter format --text <文本> --platform <平台>
```

**平台格式规范**:

| 平台        | 章节格式         | 段落格式      | 标点规范     | 特殊要求           |
| ----------- | ---------------- | ------------- | ------------ | ------------------ |
| qidian      | 第X章 + 标题     | 首行缩进2字符 | 全部中文标点 | 段末不加"。"       |
| qidian-free | 第X章 + 标题     | 首行缩进2字符 | 全部中文标点 | 同上               |
| zongheng    | Chapter X + 标题 | 顶格          | 中英标点混用 | 英文标题更易被抓取 |
| 17k         | 第X章 + 标题     | 顶格或缩进    | 全部中文标点 | 可使用部分英文标点 |
| jinjiang    | 第X章 + 标题     | 顶格          | 中文标点为主 | 严禁英文标点       |

**格式转换规则库**:

| 规则ID  | 名称         | 转换说明                 | 示例               |
| ------- | ------------ | ------------------------ | ------------------ |
| FMT-001 | 章节标题格式 | 按平台规范转换           | "第1章" → "第一章" |
| FMT-002 | 段落缩进     | 按平台要求调整首行缩进   | 缩进2字符/顶格     |
| FMT-003 | 标点符号     | 统一为平台要求的标点类型 | "，" → "，"        |
| FMT-004 | 人名格式     | 统一人名格式             | "主角" → "XXX"     |
| FMT-005 | 数字格式     | 统一数字表示             | "10000" → "一万"   |
| FMT-006 | 空格处理     | 去除/添加必要空格        | 英文前后加空格     |

**格式检查清单**:

```
□ 章节标题是否符合平台规范
□ 段落首行是否按要求缩进
□ 标点符号是否全部为中文
□ 人名/地名是否统一
□ 数字是否按要求转换
□ 敏感内容是否已处理
```

### sensitive - 敏感词

```
/nf-platform-adapter sensitive --text <文本> --platform <平台>
```

**敏感词分类体系**:

| 类别     | 说明                   | 风险等级 | 处理方式             |
| -------- | ---------------------- | -------- | -------------------- |
| 政治类   | 领导人、政党、敏感事件 | 极高     | 绝对禁止，直接删除   |
| 色情类   | 亲密关系、身体描写     | 极高     | 尺度内保留，过度删除 |
| 暴力类   | 血腥、虐待、凶杀       | 高       | 可表现但不详细       |
| 封建迷信 | 宗教、玄学、鬼神       | 中高     | 虚构题材可放宽       |
| 违法犯罪 | 赌博、毒品、诈骗       | 高       | 负面描写可保留       |
| 社会敏感 | 贫富差距、职场黑暗     | 中       | 可表现但需正向引导   |

**平台敏感词严格度对比**:

```
政治敏感度: 晋江 > 起点 > 纵横 > 17K
色情严格度: 晋江 > 起点 > 17K > 纵横
暴力容忍度: 纵横 > 17K > 起点 > 晋江
```

**敏感词处理方法**:

| 方法     | 适用场景             | 示例               |
| -------- | -------------------- | ------------------ |
| 直接删除 | 政治类、极高风险内容 | 删除涉及内容       |
| 替换     | 中高风险内容         | "小姐"→"姑娘"      |
| 隐喻     | 必要但敏感的内容     | 用"XX"代替具体器官 |
| 留白     | 不可避免的敏感场景   | 用"......"跳过     |
| 改编     | 涉及真实宗教/事件    | 虚构名称替代       |

**敏感词检测报告模板**:

```json
{
  "platform": "qidian",
  "check_date": "2024-XX-XX",
  "total_chapters": 100,
  "sensitive_words_found": 15,
  "risk_distribution": {
    "extreme_risk": 0,
    "high_risk": 3,
    "medium_risk": 5,
    "low_risk": 7
  },
  "detailed_issues": [
    {
      "chapter": 10,

    }

```

## 错误处理

| 错误码 | 说明         | 处理方式                 |
| ------ | ------------ | ------------------------ |
| PA-001 | 平台不支持   | 使用通用格式，提示不支持 |
| PA-002 | 内容为空     | 返回空结果，提示输入内容 |
| PA-003 | 敏感词过多   | 分级处理，生成修改建议   |
| PA-004 | 格式转换失败 | 保留原文，提示错误原因   |
| PA-005 | 元数据过长   | 自动截取并提示超长部分   |

**错误恢复流程**:

```
PA-001:
  - 返回通用格式版本
  - 提示该平台的specific规范

PA-002:
  - 返回格式模板
  - 提示必填字段

PA-003:
  - 列出所有敏感词及位置
  - 提供修改建议
  - 给出修改后的预览

PA-004:
  - 保留原文
  - 记录转换失败的具体位置
  - 建议手动检查

PA-005:
  - 自动截取合规长度
  - 提示截断位置和原因
  - 建议精简方案
```
