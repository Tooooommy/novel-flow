# 项目初始化技能

## 使用场景

- 用户要创建新小说项目
- 用户要初始化项目目录结构
- 用户要生成项目元数据

---

## 命令

### 项目立项

> 根据选定的创意题材, 初始化项目目录结构，生成创意报告和创作进度

```
/nf-project-init
```

**执行流程**：

**默认正文字数约为3500字/章**
**默认目标字数约200万字**

1. 获取用户选定的创意题材(包含书名、题材、篇幅、目标字数、摘要等)
2. 创建目录结构 `novels/{name}/*`, 确认正文字数、目标字数等
3. 生成创意报告：`novels/{name}/novel.md`
4. 生成创作进度：`novels/{name}/progress.md`
5. 输出完成信息

---

## 目录结构

```
novels/{name}/
├── novel.md        # 创意报告
├── progress.md     # 创作进度
├── outline/        # 大纲目录
│   ├── overview.md     # 总览
│   ├── world.md        # 世界观
│   ├── characters.md    # 人物
│   ├── plot.md          # 主线
│   ├── timeline.md      # 时间线
│   └── v-{卷号}-{卷名}.md      # 分卷大纲
├── content/        # 正文目录
│   └── v-{卷号}-{卷名}/ch-{章节号}-{章节名}.md
└── research/      # 研究目录
```

---

**novel.md 创意报告模板**：

内容包含：

- 书名、题材、标签
- 核心概念/一句话简介
- 主要角色（主角、配角、反派）
- 故事主线/核心冲突
- 世界观概述
- 金手指/设定亮点
- 目标读者/市场定位
- 预计篇幅/章节规划

**示例**：

```markdown
# 星际卖奶茶

## 基本信息

| 项目 | 内容                   |
| ---- | ---------------------- |
| 书名 | 星际卖奶茶             |
| 题材 | 科幻                   |
| 标签 | 星际、创业、甜宠、爽文 |
| 状态 | 立项                   |

## 核心概念

在星际时代，一个拥有神奇奶茶配方的年轻人，通过卖奶茶成为星际首富的故事。

## 主要角色

### 主角

- **林星河**：奶茶店老板，拥有独特的奶茶配方，性格坚韧不拔

### 配角

- **小雪**：奶茶店店员，聪明伶俐
- **老张**：星际商人，慧眼识珠

### 反派

- **黑心商人**：试图窃取配方的竞争对手

## 故事主线

1. 意外获得配方 → 开店创业
2. 遭遇竞争对手 → 智斗取胜
3. 发现星际奶茶秘密 → 事业腾飞
4. 最终成为星际奶茶大王

## 世界观概述

星际时代，人类已殖民多个星系。奶茶成为跨星系流行的饮品，但顶级奶茶配方极为稀缺。

## 金手指

- 神奇奶茶配方：能让饮品具有特殊功效（美容、增幅灵力等）
- 升级系统：随店铺等级提升解锁新配方

## 目标读者

- 喜欢科幻、创业、甜宠的读者
- 18-35岁男性为主

## 篇幅规划

- 目标字数：800万字
- 正文字数：3500字/章
- 总章节：2286章
- 分卷：23卷，每卷约99章
```

## 错误处理

| 错误码 | 说明       | 处理方式                  |
| ------ | ---------- | ------------------------- |
| PI-001 | 名称为空   | 提示用户输入小说名称      |
| PI-002 | 题材无效   | 使用默认题材（玄幻）      |
| PI-003 | 字数无效   | 提示输入有效字数（≥10万） |
| PI-004 | 目录已存在 | 询问是否覆盖或重命名      |
| PI-005 | 创建失败   | 提示权限问题或磁盘空间    |

**错误恢复流程**:

```
PI-001: 提示并要求输入小说名称
PI-002: 使用玄幻题材，继续执行
PI-003: 提示有效字数范围（10万-5000万）
PI-004:
  - 覆盖：删除旧目录，重新创建
  - 重命名：使用新名称创建
PI-005: 检查磁盘空间和目录权限
```
