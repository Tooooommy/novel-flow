# 项目初始化技能

## 使用场景

- 用户要创建新小说项目
- 用户要初始化项目目录结构
- 用户要生成项目元数据

---

## 命令

### 项目立项

> 根据选定的创意题材, 初始化项目目录结构

```
/nf-project-init
```

**执行流程**：

**默认篇幅为3500字/章**
**默认目标字数为200万**

1. 获取用户选定的创意题材(包含书名、题材、篇幅、摘要等)
2. 创建目录结构`novels/{name}/*`,
3. 生成项目文件`novels/{name}/novel.yaml`
4. 输出完成信息

---

## 目录结构

```
novels/{name}/
├── novel.yaml      # 项目元数据
├── outline/        # 大纲目录
│   ├── overview.md     # 总览
│   ├── world.md        # 世界观
│   ├── characters.md    # 人物
│   ├── plot.md          # 主线
│   ├── timeline.md      # 时间线
│   └── v-{卷号}-{卷名}.md      # 分卷大纲
├── content/        # 正文目录
│   └── v-{卷号}-{卷名}/ch-{章节号}-{章节名}.md
└── research/      # 研究目录
```

---

**novel.yaml 模板**：

内容包含：

- name: 小说名称
- genre: 题材类型
- target: 目标字数, 默认200万
- chapter_size: 每章字数, 默认3500字左右
- status: 项目状态/planning/writing/publishing
- created: 创建日期
- concept: 核心概念一句话描述
- gains: 金手指/金手指列表
- target_platforms: 目标发布平台

**示例**：

```yaml
name: 星际卖奶茶
genre: 科幻
target: 8000000
chapter_size: 3500
created: 2023-01-01
status: planning
concept: 一个关于星际卖奶茶的科幻小说
gains:
  - 金手指1
  - 金手指2
target_platforms:
  - 平台1
  - 平台2
```

## 错误处理

| 错误码 | 说明       | 处理方式                  |
| ------ | ---------- | ------------------------- |
| PI-001 | 名称为空   | 提示用户输入小说名称      |
| PI-002 | 题材无效   | 使用默认题材（玄幻）      |
| PI-003 | 字数无效   | 提示输入有效字数（≥10万） |
| PI-004 | 目录已存在 | 询问是否覆盖或重命名      |
| PI-005 | 创建失败   | 提示权限问题或磁盘空间    |

**错误恢复流程**:

```
PI-001: 提示并要求输入小说名称
PI-002: 使用玄幻题材，继续执行
PI-003: 提示有效字数范围（10万-5000万）
PI-004:
  - 覆盖：删除旧目录，重新创建
  - 重命名：使用新名称创建
PI-005: 检查磁盘空间和目录权限
```
