# 流程分析员技能

## 系统定位

```
用户 → nf-novel-flow（主控） → nf-process-analyst（流程分析）
                              └──→ 分析创作效率，识别瓶颈，改进流程
```

**核心职责**：

| 职责     | 说明                        |
| -------- | --------------------------- |
| 效率分析 | 评估各阶段时间/资源使用效率 |
| 瓶颈识别 | 发现创作流程中的阻塞点      |
| 质量追踪 | 分析返工率、质量波动原因    |
| 改进建议 | 提供针对性的优化方案        |
| 预测分析 | 估算项目完成时间和资源需求  |

---

## 命令详解

### full - 一键完整流程分析

**功能**：自动执行 analyze → bottleneck → efficiency 的完整分析流程

```
/nf-process-analyst full --logs <流程日志> [--scope <范围>] [--output <输出格式>]
```

| 参数   | 类型   | 必填 | 默认值 | 说明         |
| ------ | ------ | ---- | ------ | ------------ |
| logs   | string | 是   | -      | 流程日志路径 |
| scope  | string | 否   | full   | 分析范围     |
| output | string | 否   | md     | 输出格式     |

**执行流程**：

| 阶段                              | 步骤           | 内容                           |
| --------------------------------- | -------------- | ------------------------------ |
| **Step 1: analyze (效率分析)**    | 时间效率分析   | 统计各阶段耗时、对比预期与实际 |
|                                   | 资源效率分析   | 评估AI调用次数、Token消耗      |
|                                   | 质量效率分析   | 检测输出质量稳定性             |
|                                   | 生成效率指标   | 输出效率报告                   |
| **Step 2: bottleneck (瓶颈识别)** | 阶段耗时对比   | 定位耗时最长的阶段             |
|                                   | 阻塞点定位     | 识别导致延迟的具体环节         |
|                                   | 原因分析       | 分析瓶颈产生的根本原因         |
|                                   | 瓶颈等级划分   | 区分关键瓶颈和次要瓶颈         |
| **Step 3: efficiency (效率评估)** | 综合效率评分   | 计算整体创作效率得分           |
|                                   | 对标分析       | 与标准效率指标对比             |
|                                   | 改进优先级排序 | 按影响程度排列优化顺序         |

**分析范围选项**：

| 范围   | 说明            | 适用场景         |
| ------ | --------------- | ---------------- |
| full   | 完整流程分析    | 项目结束复盘     |
| phase  | 指定阶段分析    | 某阶段效率异常   |
| task   | 单任务分析      | 特定任务执行分析 |
| recent | 最近N次创作分析 | 快速诊断         |

**示例**：

```bash
/nf-process-analyst full --logs ./logs/逆天剑尊-流程日志.md
/nf-process-analyst full --logs ./logs --scope phase --output json
```

### analyze - 流程效率分析

**功能**：深入分析创作流程各阶段的效率指标

```
/nf-process-analyst analyze --logs <流程日志> [--metrics <指标列表>] [--granularity <粒度>]
```

| 参数        | 类型   | 必填 | 默认值 | 说明         |
| ----------- | ------ | ---- | ------ | ------------ |
| logs        | string | 是   | -      | 流程日志路径 |
| metrics     | string | 否   | all    | 指标列表     |
| granularity | string | 否   | phase  | 分析粒度     |

**分析指标详解**：

| 指标类型 | 指标名称     | 计算方式                | 优秀标准      |
| -------- | ------------ | ----------------------- | ------------- |
| 时间效率 | 阶段周期时间 | 结束时间 - 开始时间     | < 预期        |
| 时间效率 | 创作速度     | 完成字数 / 实际耗时     | > 2000字/小时 |
| 时间效率 | 有效创作占比 | 实际创作时间 / 总耗时   | > 70%         |
| 资源效率 | AI调用次数   | 总AI调用次数            | 越少越好      |
| 资源效率 | 优化轮次     | 平均章节优化次数        | < 2次         |
| 质量效率 | 一次通过率   | 初审通过数 / 总章节数   | > 80%         |
| 质量效率 | 返工率       | 返工章节数 / 总章节数   | < 20%         |
| 质量效率 | 质量波动系数 | 质量评分标准差 / 平均分 | < 0.15        |

**输出示例**：

```
【效率分析报告】

1. 时间效率
   - 立项阶段: 2小时 (预期: 3小时) ✓
   - 大纲阶段: 8小时 (预期: 6小时) ✗ 超时33%
   - 创作阶段: 120小时 (预期: 100小时) ✗ 超时20%
   - 优化阶段: 40小时 (预期: 30小时) ✗ 超时33%

2. 资源效率
   - AI调用次数: 平均每章3.2次
   - 优化轮次: 平均每章1.8次
   - 断点续传: 使用率85%

3. 质量效率
   - 一次通过率: 75%
   - 返工章节: 6章 (共30章)
   - 质量问题分布:
     * 逻辑矛盾: 3次
     * AI味道: 2次
     * 节奏问题: 1次
```

**示例**：

```bash
/nf-process-analyst analyze --logs ./logs/逆天剑尊.md --metrics 时间效率,质量效率
/nf-process-analyst analyze --logs ./logs --granularity chapter
```

### bottleneck - 识别瓶颈

**功能**：识别创作流程中的瓶颈点，分析原因

```
/nf-process-analyst bottleneck --timeline <阶段时间线> [--threshold <阈值>] [--root-cause]
```

| 参数       | 类型   | 必填 | 默认值 | 说明       |
| ---------- | ------ | ---- | ------ | ---------- |
| timeline   | string | 是   | -      | 阶段时间线 |
| threshold  | number | 否   | 1.5    | 瓶颈阈值   |
| root-cause | flag   | 否   | false  | 原因分析   |

**瓶颈识别算法**：

```
┌────────────────────────────────────────────────────────────────────────┐
│                        瓶颈识别判断逻辑                                  │
├────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  瓶颈等级 = 实际耗时 / 预期耗时                                          │
│                                                                         │
│  ├── 等级1 (1.0-1.3):  正常波动                                         │
│  ├── 等级2 (1.3-1.5):  轻度瓶颈  → 建议关注                             │
│  ├── 等级3 (1.5-2.0):  中度瓶颈  → 需要改进                            │
│  └── 等级4 (>2.0):     严重瓶颈  → 必须解决                             │
│                                                                         │
└────────────────────────────────────────────────────────────────────────┘
```

**瓶颈分类**：

| 瓶颈类型 | 特征                 | 常见原因               |
| -------- | -------------------- | ---------------------- |
| 时间瓶颈 | 阶段耗时远超预期     | 难度评估不足、返工过多 |
| 资源瓶颈 | 等待AI响应、人力不足 | 并发限制、依赖阻塞     |
| 质量瓶颈 | 反复返工、质量不稳定 | 标准不清、能力不足     |
| 协作瓶颈 | 阶段间衔接不畅       | 沟通不足、交接不清     |
| 创意瓶颈 | 构思卡壳、反复修改   | 创意枯竭、方向不明     |

**根因分析框架**：

```
┌────────────────────────────────────────────────────────────────────────┐
│                        5Why 根因分析法                                  │
├────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  问题: 大纲阶段耗时超标100%                                             │
│                                                                         │
│  Why1: 为什么大纲阶段超时？                                             │
│  → 因为情节设计反复修改                                                 │
│                                                                         │
│  Why2: 为什么情节设计反复修改？                                         │
│  → 因为人物设定中途改变                                                 │
│                                                                         │
│  Why3: 为什么人物设定改变？                                             │
│  → 因为初期人物性格设计不合理                                           │
│                                                                         │
│  Why4: 为什么人物性格设计不合理？                                       │
│  → 因为缺乏人物原型参考                                                 │
│                                                                         │
│  Why5: 为什么没有人物原型参考？                                         │
│  → 因为知识库中没有相关经验沉淀                                          │
│                                                                         │
│  根因: 知识库经验不足 → 建议: 丰富人物设定模板库                         │
│                                                                         │
└────────────────────────────────────────────────────────────────────────┘
```

**输出示例**：

```
【瓶颈识别报告】

1. 严重瓶颈 (等级4)
   瓶颈点: 大纲阶段
   耗时: 16小时 (预期: 6小时)
   类型: 创意瓶颈
   根因: 情节设计反复修改

2. 中度瓶颈 (等级3)
   瓶颈点: 优化阶段
   耗时: 40小时 (预期: 25小时)
   类型: 质量瓶颈
   根因: AI味道问题反复出现

3. 改进优先级
   [1] 优化人物设定流程 (预计提升30%效率)
   [2] 建立AI味道检测标准 (预计减少50%优化轮次)
```

**示例**：

```bash
/nf-process-analyst bottleneck --timeline ./timeline.md --threshold 1.5 --root-cause
```

### efficiency - 效率评估

**功能**：综合评估创作效率，计算效率评分

```
/nf-process-analyst efficiency --data <流程数据> [--compare <基准>] [--weight <权重>]
```

| 参数    | 类型   | 必填 | 默认值    | 说明     |
| ------- | ------ | ---- | --------- | -------- |
| data    | string | 是   | -         | 流程数据 |
| compare | string | 否   | benchmark | 对标基准 |
| weight  | string | 否   | default   | 权重配置 |

**效率评分体系**：

| 评分范围 | 等级 | 说明         | 建议       |
| -------- | ---- | ------------ | ---------- |
| 90-100   | 优秀 | 效率远超预期 | 保持现状   |
| 75-89    | 良好 | 效率高于平均 | 细节优化   |
| 60-74    | 中等 | 效率处于平均 | 针对性改进 |
| 45-59    | 较差 | 效率低于平均 | 系统性改进 |
| <45      | 很差 | 效率严重不足 | 全面重构   |

**效率评分计算公式**：

```
效率评分 = Σ(指标得分 × 权重) / Σ权重

指标得分计算：
  得分 = 100 - (实际值 - 目标值) / 目标值 × 100
  封顶: 0-100

默认权重配置：
  时间效率: 30%
  质量效率: 35%
  资源效率: 20%
  创意效率: 15%
```

**对标基准选项**：

| 基准类型  | 说明           | 数据来源            |
| --------- | -------------- | ------------------- |
| benchmark | 行业平均标准   | 历史项目统计        |
| internal  | 内部最优/平均  | 团队历史最佳/平均值 |
| self      | 与自身历史对比 | 项目自身纵向对比    |
| target    | 与目标值对比   | 用户设定的预期目标  |

**输出示例**：

```
【效率评估报告】

综合效率评分: 72/100 (中等)

各维度得分:
├── 时间效率: 65/100 (得分率 65%)
│   ├── 立项: 90% (优秀)
│   ├── 大纲: 45% (较差)
│   ├── 创作: 70% (中等)
│   └── 优化: 55% (较差)
│
├── 质量效率: 78/100 (得分率 78%)
│   ├── 一次通过率: 75%
│   └── 返工率: 25%
│
├── 资源效率: 80/100 (得分率 80%)
│   ├── AI调用效率: 良好
│   └── 断点使用: 85%
│
└── 创意效率: 70/100 (得分率 70%)
    └── 创意稳定性: 中等

对标分析:
├── vs 行业基准: +5% (优于平均)
├── vs 内部平均: -8% (低于平均)
└── vs 自身目标: -15% (未达目标)

改进建议:
[高优先级] 优化大纲阶段流程 (预计+12分)
[中优先级] 建立AI味道检测标准 (预计+8分)
```

**示例**：

```bash
/nf-process-analyst efficiency --data ./data/project.json --compare internal
```

### report - 生成分析报告

**功能**：生成完整的流程分析报告

```
/nf-process-analyst report --project <项目> [--scope <范围>] [--template <模板>] [--output <格式>]
```

| -------- | ------ | ---- | -------- | -------- |
| project  | string | 是   | -        | 项目名称 |
| scope    | string | 否   | full     | 报告范围 |
| template | string | 否   | standard | 报告模板 |
| output   | string | 否   | md       | 输出格式 |

### suggest - 改进建议

**功能**：基于问题分析提供针对性改进建议

```
/nf-process-analyst suggest --issues <问题列表> [--constraints <约束>] [--priority <优先级>]
```

| 参数        | 类型   | 必填 | 默认值 | 说明       |
| ----------- | ------ | ---- | ------ | ---------- |
| issues      | string | 是   | -      | 问题列表   |
| constraints | string | 否   | none   | 约束条件   |
| priority    | string | 否   | impact | 优先级标准 |

**改进建议框架**：

```
┌────────────────────────────────────────────────────────────────────────┐
│                        改进建议优先级框架                                │
├────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  优先级 = 影响力 × 紧迫性 / 实施成本                                   │
│                                                                         │
│  高优先级: 高影响力 + 高紧迫性 + 低成本                                 │
│  中优先级: 中影响力 + 中紧迫性 + 中成本                                 │
│  低优先级: 低影响力 + 低紧迫性 + 高成本                                 │
│                                                                         │
└────────────────────────────────────────────────────────────────────────┘
```

**建议类型**：

| 类型      | 说明          | 预期效果 |
| --------- | ------------- | -------- |
| process   | 流程优化      | 减少耗时 |
| template  | 模板/标准制定 | 减少返工 |
| tool      | 工具改进      | 提升效率 |
| skill     | 能力提升      | 长期受益 |
| knowledge | 知识沉淀      | 复用经验 |

**改进建议示例库**：

| 问题             | 建议类型  | 具体建议               | 预期提升 |
| ---------------- | --------- | ---------------------- | -------- |
| 大纲阶段超时     | process   | 增加大纲预审环节       | 30%      |
| AI味道问题       | template  | 建立AI味道检测标准清单 | 50%      |
| 优化轮次过多     | tool      | 引入批量优化模式       | 40%      |
| 人物设定反复修改 | knowledge | 丰富人物原型库         | 35%      |
| 情节逻辑矛盾     | skill     | 培训逻辑检查方法       | 45%      |

**约束条件处理**：

| 约束类型 | 说明     | 处理方式                 |
| -------- | -------- | ------------------------ |
| time     | 时间约束 | 优先选择快速见效的建议   |
| budget   | 预算约束 | 优先选择低成本建议       |
| skill    | 技能约束 | 优先选择容易实施的建议   |
| resource | 资源约束 | 优先选择资源需求低的建议 |

## 分析维度详解

### 时间效率分析

| 指标         | 定义                  | 测量方式      |
| ------------ | --------------------- | ------------- |
| 阶段周期时间 | 阶段开始到结束的时长  | 时间戳计算    |
| 有效创作时间 | 实际用于创作的时间    | 去除休息/等待 |
| 创作速度     | 单位时间完成的字数    | 字数/时间     |
| 等待时间     | 等待AI响应/反馈的时间 | 时间戳计算    |
| 中断次数     | 创作过程中断的次数    | 日志统计      |

### 资源效率分析

| 指标       | 定义                 | 测量方式          |
| ---------- | -------------------- | ----------------- |
| AI调用效率 | 每次AI调用的有效输出 | 输出质量/调用次数 |
| 模板复用率 | 使用已有模板的比例   | 模板使用/总创作   |
| 断点续传率 | 使用断点续传的比例   | 续传次数/中断次数 |
| 知识复用率 | 复用已有知识的比例   | 复用次数/总创作   |

### 质量效率分析

| 指标         | 定义               | 测量方式        |
| ------------ | ------------------ | --------------- |
| 一次通过率   | 初审即通过的比例   | 通过数/总提交数 |
| 返工率       | 需要返工的比例     | 返工数/总提交数 |
| 质量波动系数 | 质量评分的离散程度 | 标准差/平均值   |
| 问题密度     | 单位字数的问题数   | 问题数/总字数   |

### 协作效率分析

| 指标         | 定义                 | 测量方式        |
| ------------ | -------------------- | --------------- |
| 交接顺畅度   | 阶段间衔接的流畅程度 | 交接等待时间    |
| 指令清晰度   | 用户指令的明确程度   | 澄清次数/指令数 |
| 反馈响应速度 | 对用户反馈的响应时间 | 时间戳计算      |

## 指标体系

### 核心指标 (KPIs)

| 指标       | 目标值     | 警告值     | 严重值     |
| ---------- | ---------- | ---------- | ---------- |
| 创作速度   | >2000字/时 | <1500字/时 | <1000字/时 |
| 一次通过率 | >80%       | <60%       | <40%       |
| 返工率     | <15%       | >25%       | >40%       |
| AI调用效率 | >500字/次  | <300字/次  | <200字/次  |
| 阶段准时率 | >85%       | <70%       | <50%       |

### 指标计算公式

```
创作速度 = 完成字数 / 实际创作时间

有效创作时间 = 总耗时 - 等待时间 - 休息时间 - 中断时间

AI调用效率 = AI输出字数 / AI调用次数

一次通过率 = 初审通过数 / 提交总数 × 100%

返工率 = 返工章节数 / 总章节数 × 100%

阶段准时率 = 准时完成阶段数 / 总阶段数 × 100%
```

## 错误处理

| 错误码 | 说明           | 处理方式                   |
| ------ | -------------- | -------------------------- |
| PA-001 | 日志文件不存在 | 提示检查路径               |
| PA-002 | 日志格式错误   | 显示正确格式示例           |
| PA-003 | 数据不完整     | 使用可用数据，标注缺失项   |
| PA-004 | 分析超时       | 返回部分结果，标注未完成项 |
| PA-005 | 基准数据缺失   | 使用默认基准或跳过对比     |
| PA-006 | 计算错误       | 检查数据，报告具体错误     |

**错误恢复流程**：

```
PA-001: 检查 --logs 路径是否正确
      → 使用 ls 或 dir 检查文件是否存在
      → 确认文件格式是否正确

PA-002: 显示标准日志格式
      → 提供日志模板
      → 建议修复现有日志

PA-003: 使用可用数据继续分析
      → 在报告中标注数据缺失项
      → 建议补充数据

PA-005: 使用默认基准
      → 行业平均值为基准
      → 在报告中说明
```

## 使用示例

```bash
/nf-process-analyst full --logs ./logs/逆天剑尊.md --output md > 复盘报告.md
```

**场景2：针对性分析**

```bash
/nf-process-analyst analyze --logs ./logs/逆天剑尊.md --metrics 时间效率
/nf-process-analyst bottleneck --timeline ./timeline.md --root-cause
```

**场景3：效率评估**

```bash
/nf-process-analyst efficiency --data ./data/project.json --compare internal
```

**场景4：生成报告**

```bash
/nf-process-analyst report --project 逆天剑尊 --template detailed --output html > 报告.html
```

**场景5：获取改进建议**

```bash
/nf-process-analyst suggest --issues "大纲超时,AI味道问题" --constraints time=2周
```
