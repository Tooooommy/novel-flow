# 专业优化技能

## 使用场景

- 用户要优化章节内容
- 用户觉得文章有AI味道
- 用户要优化对话
- 用户要优化战斗描写
- 用户要优化卷间衔接

---

## 命令

### full - 全量优化

```
/nf-specialist-optimizer full [--chapter <章节号>] [--volume <卷号>]
```

| 参数    | 类型   | 必填 | 默认值 | 说明   |
| ------- | ------ | ---- | ------ | ------ |
| chapter | number | 否   | 当前   | 章节号 |
| volume  | number | 否   | 当前   | 分卷号 |

**执行流程**:

```
┌─────────────────────────────────────────────────────────────────────┐
│                      full 命令执行流程                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. style (文风优化)                                                  │
│     │ → 去AI味道、统一文风                                            │
│     │                                                                │
│     └──→ 2. plot (情节优化)                                          │
│               │ → 逻辑矛盾、节奏调整、爽点密度                          │
│               │                                                       │
│               └──→ 3. dialogue (对话优化)                             │
│                         │ → 口语化、潜台词、人物性格                     │
│                         │                                              │
│                         └──→ 4. battle (战斗优化)                      │
│                                   │ → 动作紧凑、战术清晰                  │
│                                   │                                     │
│                                   └──→ 5. connection (衔接优化)          │
│                                             │ → 卷间连贯性              │
│                                             ↓                          │
│                                       优化完成                          │
└─────────────────────────────────────────────────────────────────────┘
```

**执行方案**:

| 步骤          | 操作                         | 产出       |
| ------------- | ---------------------------- | ---------- |
| 1. style      | 去AI味道、统一文风           | 文风一致稿 |
| 2. plot       | 检查逻辑、调整节奏、优化爽点 | 情节优化稿 |
| 3. dialogue   | 优化对话、添加潜台词         | 对话优化稿 |
| 4. battle     | 紧凑动作、清晰战术           | 战斗优化稿 |
| 5. connection | 衔接优化、状态统一           | 最终稿     |

## 内部命令

### style - 文风优化

```
/nf-specialist-optimizer style [--chapter <章节号>]
```

**执行流程**:

```
原文 → 识别AI特征 → 针对性优化 → 文风统一稿
```

**AI特征识别与优化**:

| AI特征     | 识别模式                 | 优化方法                   |
| ---------- | ------------------------ | -------------------------- |
| 过度解释   | "他很生气，因为他被骗了" | 删除解释，保留"他握紧拳头" |
| 机械化副词 | "缓缓地"、"轻轻地"       | 删除或替换                 |
| 重复句式   | "......"                 | 变化句式长度               |
| 情绪词堆砌 | "非常愤怒、极度生气"     | 用身体反应替代             |
| 对话直白   | "我很生气！"             | 增加潜台词                 |

**去AI味道执行方案**:

| 步骤 | 操作               | 产出     |
| ---- | ------------------ | -------- |
| 1    | 识别原文中的AI特征 | 问题列表 |
| 2    | 删除过度解释性词语 | 简洁表达 |
| 3    | 替换机械化副词     | 自然表达 |
| 4    | 情绪用身体反应表现 | 具象化   |
| 5    | 对话口语化         | 自然对话 |
| 6    | 长短句交错         | 节奏感   |

**示例**:

```
【原文(AI味)】
他缓缓地睁开眼睛，心中充满了惊讶。他轻轻地坐起身来，回忆着刚才发生的一切。

【优化后(去AI味)】
他睁开眼。
记忆还停留在那一剑刺来的瞬间。
```

### plot - 情节优化

```
/nf-specialist-optimizer plot [--chapter <章节号>]
```

**执行流程**:

```
原文 → 逻辑检查 → 节奏分析 → 爽点评估 → 优化稿
```

**逻辑矛盾检查**:

| 检查项   | 示例                         | 解决方案       |
| -------- | ---------------------------- | -------------- |
| 战力崩坏 | 前章被秒杀，后章秒杀对方     | 调整实力对比   |
| 时间矛盾 | A在X地说完话，B立刻在Y地回应 | 增加过渡或调整 |
| 物品矛盾 | 前面说丢失，后面突然出现     | 删除或解释来源 |
| 性格矛盾 | 人物行为与性格设定不符       | 修正行为       |
| 常识矛盾 | 物理/逻辑违反设定            | 调整描写       |

**节奏调整方法**:

| 问题     | 表现                   | 解决方案           |
| -------- | ---------------------- | ------------------ |
| 节奏过慢 | 大段描写、内心独白过多 | 删除冗余，增加冲突 |
| 节奏过快 | 情节跳跃、缺乏过渡     | 增加铺垫，丰富细节 |
| 高潮不足 | 平铺直叙               | 增加反转或爽点     |

**爽点密度评估**:

```
章节爽点分布:
  - 章初: 期待感铺垫
  - 章中: 小爽点1-2个
  - 章中后: 中爽点1个
  - 章末: 大爽点/悬念

优化目标: 每3000字至少1个有效爽点
```

### dialogue - 对话优化

```
/nf-specialist-optimizer dialogue [--chapter <章节号>]
```

**执行流程**:

```
原文对话 → 人物性格分析 → 潜台词设计 → 节奏调整 → 优化稿
```

**对话优化方案**:

| 步骤 | 操作                         | 产出       |
| ---- | ---------------------------- | ---------- |
| 1    | 提取原文所有对话             | 对话列表   |
| 2    | 分析每句对话的角色性格符合度 | 问题点     |
| 3    | 设计潜台词和言外之意         | 潜台词设计 |
| 4    | 调整对话节奏（长短句搭配）   | 节奏优化   |
| 5    | 替换不符合性格的表达         | 性格统一   |

**对话优化示例**:

```
【原文】
"你好厉害啊！"小美说。
"没什么，只是运气好。"主角回答。

【优化后】
小美眼睛亮晶晶的："你这也太强了吧！"
主角耸耸肩，嘴角带着一丝不易察觉的笑意："运气好而已。"
小美凑近了些："骗谁呢，你当我眼瞎？"
```

### battle - 战斗优化

```
/nf-specialist-optimizer battle [--chapter <章节号>]
```

**执行流程**:

```
战斗段落 → 强度评估 → 动作分析 → 战术设计 → 紧张感渲染 → 优化稿
```

**战斗优化方案**:

| 步骤 | 操作                 | 产出     |
| ---- | -------------------- | -------- |
| 1    | 提取战斗段落         | 战斗片段 |
| 2    | 评估战斗强度等级     | 强度定级 |
| 3    | 分析动作描写的精准度 | 动作问题 |
| 4    | 设计战术和策略       | 战术优化 |
| 5    | 渲染紧张氛围         | 氛围增强 |

**战斗描写模板**:

| 强度      | 结构                       | 要素              |
| --------- | -------------------------- | ----------------- |
| 低(1级)   | 结果+2-3动作               | 一招制胜/快速结束 |
| 中(2级)   | 攻防转换+5-8动作           | 有来有回/试探     |
| 高(3级)   | 招式名+动作细节+10+动作    | 苦战/技能展示     |
| 决战(4级) | 多个回合+心理+环境+20+动作 | 生死战/逆转       |

**优化示例**:

```
【原文】
两人打了起来，最后主角赢了。

【高强度优化】
剑光如雪。
"破天九式——第七式！"
主角双瞳赤红，体内灵力狂涌。剑芒暴涨三丈，化作一道银虹，直取对手咽喉。
对手面色剧变，连退数步，却仍被剑气所伤。
鲜血飞溅。
"你......"对手捂着伤口，眼中满是不可置信。
主角收剑而立，衣袂飘飘："还有谁？"
```

### connection - 卷间衔接

```
/nf-specialist-optimizer connection [--volume <卷号>]
```

**执行流程**:

```
上卷结尾 → 衔接分析 → 情节连贯 → 人物状态 → 文风统一 → 优化稿
```

**衔接优化方案**:

| 步骤 | 操作                   | 产出       |
| ---- | ---------------------- | ---------- |
| 1    | 分析上卷结尾的关键信息 | 承接要点   |
| 2    | 检查下卷开头的承接     | 连贯性评估 |
| 3    | 优化情节衔接           | 流畅过渡   |
| 4    | 统一人物状态           | 状态一致   |
| 5    | 统一文风               | 文风连贯   |

**衔接检查清单**:

```
□ 人物状态: 伤势、能力、情绪是否延续
□ 物品道具: 重要物品是否携带/保管
□ 剧情进度: 悬念是否解答/延续
□ 时间线: 间隔多久、是否合理
□ 势力关系: 各方态度是否有变
□ 世界观: 规则/设定是否一致
```

## 错误处理

| 错误码 | 说明       | 处理方式   |
| ------ | ---------- | ---------- |
| SO-001 | 章节不存在 | 跳过优化   |
| SO-002 | 卷号无效   | 使用当前卷 |
| SO-003 | 优化失败   | 返回原文   |
| SO-004 | AI味道过重 | 多次优化   |
