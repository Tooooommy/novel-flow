# 分卷管理技能

## 使用场景

- 用户要生成分卷大纲
- 用户要管理分卷状态
- 用户要拆分总大纲

---

## 命令

### full - 生成分卷大纲

```
/nf-volume-manager full [--volumes <卷数>]
/nf outline [--volumes <卷数>]
/nf volume [--volumes <卷数>]
```

**执行流程**:

```
总大纲 → split(拆分) → create(创建) → plan(规划)
```

**分卷结构设计标准**:

| 阶段             | 维度       | 要点                                                                                                   |
| ---------------- | ---------- | ------------------------------------------------------------------------------------------------------ |
| **S1: 卷数确定** | 字数预估法 | 总字数 = 卷数 × 每卷章数 × 每章字数；短篇(30-60万)/中篇(60-100万)/长篇(100-300万)                      |
|                  | 剧情阶段法 | 例: 觉醒→成长→小高潮→挫折→崛起→高潮 = 6卷                                                              |
|                  | 市场适应法 | 起点: 3000字/天，100万字上架；晋江: 2000字/天，VIP需30万字                                             |
| **S2: 每卷章数** | 建议范围   | 30-60章/卷，最优40-50章/卷                                                                             |
|                  | 分配原则   | 首卷25-35章(引入期)；中间卷40-50章(高潮积累)；终卷35-45章(收尾)                                        |
| **S3: 卷主题**   | 主题设计   | 每卷独立主题；贯穿情感线                                                                               |
|                  | 主题示例   | 卷1:觉醒/困境；卷2:成长/初试锋芒；卷3:崛起/小高潮；卷4:危机/挫折；卷5:觉醒/再崛起；卷6:最终对决/大高潮 |
| **S4: 章节规划** | 字数标准   | 单章2000-3000字                                                                                        |
|                  | 章内结构   | 开篇钩子→铺垫(300-500字)→发展(1000-1500字)→高潮/转折(500-800字)→结尾悬念(100-200字)                    |
|                  | 类型分布   | 推进型60%(主线)；铺垫型25%(世界观/情感)；高潮型15%(大场面/转折)                                        |

**分卷主题模板库**:

| 卷号 | 主题类型        | 核心内容                       | 爽点类型           | 章数建议 |
| ---- | --------------- | ------------------------------ | ------------------ | -------- |
| 1    | 觉醒/困境开局   | 主角出场、困境设定、金手指觉醒 | 身份反转、扮猪吃虎 | 25-35章  |
| 2    | 成长/初试锋芒   | 首次大战、势力认知、实力积累   | 小规模胜利、打脸   | 35-45章  |
| 3    | 崛起/小高潮     | 势力成型、地位提升、名声传播   | 越级挑战、势力扩张 | 40-50章  |
| 4    | 危机/挫折       | 强大敌人、失败、低谷期         | 绝境求生、隐藏实力 | 40-50章  |
| 5    | 觉醒/再崛起     | 领悟、突破、反转               | 复仇反击、实力认证 | 40-50章  |
| 6    | 最终对决/大高潮 | 终极boss、胜利、完美结局       | 终极对决、圆满收尾 | 35-45章  |

**输出**:

```
novels/{project}/
├── outline.md
├── volume-1.md
├── volume-2.md
└── volume-3.md
```

## 内部命令

### split - 拆分大纲

```
/nf-volume-manager split [--volumes <卷数>]
/nf outline split [--volumes <卷数>]
```

**拆分原则**:

| 拆分维度 | 操作                 | 标准                |
| -------- | -------------------- | ------------------- |
| 剧情阶段 | 按起承转合划分卷数   | 每卷一个完整阶段    |
| 时间线   | 按时间跳跃点划分     | 需有过渡说明        |
| 空间线   | 按地点变更划分       | 每卷1-2个主要场景   |
| 人物线   | 按人物关系变化划分   | 情感/势力重大变化点 |
| 高潮分布 | 确保每卷有独立小高潮 | 卷末应有悬念        |

**拆分检查清单**:

```
□ 每卷是否都有独立主题
□ 每卷是否都有完整的三幕结构(开头/中段/结尾)
□ 卷与卷之间是否有明确承接
□ 高潮是否合理分布(避免前卷过高导致后卷乏力)
□ 情感线是否贯穿全局
□ 伏笔是否在合适时机回收
```

### create - 创建分卷

```
/nf-volume-manager create [--volume-num <序号>]
/nf outline create [--volume-num <序号>]
```

**分卷创建标准**:

| 组成部分 | 内容要求                 | 字数建议  |
| -------- | ------------------------ | --------- |
| 卷标题   | 简洁、有吸引力、体现主题 | 4-10字    |
| 卷简介   | 本卷核心内容概述         | 100-200字 |
| 核心冲突 | 本卷主要矛盾             | 1-2句话   |
| 主要人物 | 本卷出场的重要人物       | 列出清单  |
| 场景设置 | 本卷主要场景             | 1-3个     |
| 章节目录 | 各章标题和小节点         | 40-50章   |

**分卷文档模板**:

```markdown
# 第X卷: 《卷标题》

## 卷简介

本卷核心内容概述，承接上卷、开启本卷、留下悬念。

## 核心冲突

本卷主要矛盾和看点。

## 主要人物

- 主角: 本卷主要行动
- 女主1: 本卷情感发展
- 反派: 本卷威胁来源

## 场景设置

- 场景1: 地点和特点
- 场景2: 地点和特点

## 章节目录

### 第一阶段: 铺垫期 (第X-X章)

### 第二阶段: 发展期 (第X-X章)

### 第三阶段: 高潮期 (第X-X章)
```

### status - 分卷状态

```
/nf-volume-manager status [--project <项目>]
/nf outline status [--project <项目>]
```

**状态跟踪标准**:

| 状态字段   | 说明                  | 更新时机       |
| ---------- | --------------------- | -------------- |
| 进度       | 已完成章节数/总章节数 | 每完成一章     |
| 字数       | 已完成字数/目标字数   | 每完成一章     |
| 质量评分   | 当前卷质量评估        | 每次优化后     |
| 高潮完成度 | 本卷高潮是否完成      | 高潮章节完成时 |
| 情感线进度 | 本卷情感推进程度      | 情感戏完成时   |
| 伏笔埋入   | 本卷埋下的伏笔数量    | 伏笔埋入时     |
| 伏笔回收   | 本卷回收的伏笔数量    | 伏笔回收时     |

## 分卷质量评估标准

| 评估维度   | 权重 | 评分标准                   | 达标分数 |
| ---------- | ---- | -------------------------- | -------- |
| 结构完整性 | 25%  | 有开头、发展、高潮、结尾   | ≥8/10    |
| 节奏把控   | 20%  | 无过慢或过快章节           | ≥7/10    |
| 高潮设计   | 20%  | 每卷有独立高潮且与整体呼应 | ≥8/10    |
| 人物成长   | 15%  | 主角有明显成长轨迹         | ≥7/10    |
| 伏笔布局   | 10%  | 埋线自然、回收合理         | ≥6/10    |
| 情感推进   | 10%  | 情感线有进展、不拖沓       | ≥7/10    |

## 错误处理

| 错误码 | 说明     | 处理方式                  |
| ------ | -------- | ------------------------- |
| VM-001 | 大纲为空 | 使用默认结构(3卷标准模板) |
| VM-002 | 卷数无效 | 默认3卷                   |
| VM-003 | 章节超额 | 自动调整或提示拆分        |
| VM-004 | 主题重复 | 提示重新设计卷主题        |

**错误恢复流程**:

```
VM-001: 加载默认3卷结构模板，按模板引导用户补充
VM-002: 询问用户目标字数，重新推荐卷数
VM-003:
  - 若超额<10%: 调整章节分配
  - 若超额>10%: 提示拆分为更多卷
VM-004: 分析现有卷主题，提供差异化建议
```
