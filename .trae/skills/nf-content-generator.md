# 内容生成技能

## 使用场景

- 用户要生成章节正文
- 用户要创作特定场景
- 用户要编写对话
- 用户要生成动作/战斗场景
- 用户要生成情感场景

---

## 命令

### full - 生成章节正文

```
/nf-content-generator full --volume <卷号> --chapter <章节号> [--previous <上章摘要>]
```

| 参数     | 类型   | 必填 | 默认值 | 说明     |
| -------- | ------ | ---- | ------ | -------- |
| volume   | number | 是   | -      | 分卷号   |
| chapter  | number | 是   | -      | 章节号   |
| previous | string | 否   | -      | 上章摘要 |

**执行流程**:

```
┌─────────────────────────────────────────────────────────────────────┐
│                     full 命令执行流程                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. pacing (节奏设计)                                                │
│     │ → 确定本章节奏类型：铺垫/冲突/高潮/转折                           │
│     │                                                                │
│     └──→ 2. scene (生成场景)                                         │
│               │ → 根据大纲和节奏生成场景布置                           │
│               │                                                        │
│               └──→ 3. dialogue (生成对话)                              │
│                         │ → 生成角色对话                               │
│                         │                                              │
│                         └──→ 4. description (生成描写)                  │
│                                   │ → 场景/人物/环境描写                │
│                                   │                                    │
│                                   └──→ 5. action (生成动作)              │
│                                             │ → 动作/战斗场景            │
│                                             │                          │
│                                             └──→ 6. emotion (生成情感)    │
│                                                       │ → 情感渲染       │
│                                                       │                │
│                                                       ↓                │
│                                                 整合输出章节            │
└─────────────────────────────────────────────────────────────────────┘
```

**执行方案**:

| 步骤           | 操作                     | 产出     |
| -------------- | ------------------------ | -------- |
| 1. pacing      | 确定本章节奏类型和目标   | 节奏计划 |
| 2. scene       | 生成场景布置、背景、道具 | 场景要素 |
| 3. dialogue    | 生成角色对话（含潜台词） | 对话内容 |
| 4. description | 生成环境、人物、氛围描写 | 描写片段 |
| 5. action      | 生成动作描写、战斗动作   | 动作片段 |
| 6. emotion     | 生成情感渲染、内心独白   | 情感片段 |

**智能行为**:

- 自动衔接上章结尾
- 自动去AI味道
- 节奏控制

## 内部命令

### pacing - 节奏设计

```
/nf-content-generator pacing --outline <大纲> --type <类型>
```

**执行方案**:

| 步骤 | 操作                                | 产出     |
| ---- | ----------------------------------- | -------- |
| 1    | 分析本章在大纲中的位置              | 章节类型 |
| 2    | 确定节奏目标（铺垫/冲突/高潮/转折） | 节奏类型 |
| 3    | 规划爽点出现位置                    | 爽点计划 |
| 4    | 生成节奏曲线                        | 节奏数据 |

**节奏类型定义**:

| 类型 | 说明                 | 适用场景 |
| ---- | -------------------- | -------- |
| 铺垫 | 引入新元素、埋下伏笔 | 章初     |
| 冲突 | 矛盾升级、压力增大   | 章中     |
| 高潮 | 爽点爆发、情节巅峰   | 章中后   |
| 转折 | 意外发生、局势逆转   | 章末     |
| 收尾 | 解决当前、承接下章   | 章末     |

**节奏曲线示例**:

```
情感强度
    │
 5.0 │                              ★ 高潮
    │           ★ 高潮              (逆转)
 4.0 │      ★ 高潮    /           /
    │   ★ 高潮    / ★ 转折       /
 3.0 │      ↗ ↘  /            ★ 铺垫
    │   ★ 铺垫  /
 2.0 │
    │★ 铺垫
 1.0 │
    └──────────────────────────────→ 章节进度
      10%  20%  30%  40%  50%  60%  70%  80%  90%
```

**节奏量化模型**:

情感曲线计算:

- 基础值: 极度负面(-1.0) → 极度正面(+1.0)
- 情感词权重: 强1.5x / 中1.0x / 弱0.5x
- 转折词(但是/然而): 方向反转×0.8
- 递进词(更/尤其): 当前值×1.2

爽点密度公式: `S = (Σ爽点权重) / 字数 × 1000`

- 逆转爽点(权重1.5): 反杀、打脸、没想到
- 升级爽点(权重1.3): 突破、晋升、觉醒
- 情报爽点(权重1.2): 原来、真相、秘密

张力指数: `T = 0.3×冲突 + 0.25×悬念 + 0.2×时间压力 + 0.15×风险度 + 0.1×不确定性`

**速度等级**:

| 等级 | 字数/场景 | 短句比例 | 内心描写 |
| ---- | --------- | -------- | -------- |
| 极快 | 500-800   | ≥70%     | ≤10%     |
| 快   | 800-1200  | 50-70%   | 10-20%   |
| 中   | 1200-2000 | 30-50%   | 20-30%   |
| 慢   | 2000-3000 | 20-30%   | 30-50%   |
| 极慢 | >3000     | <20%     | >50%     |

### scene - 生成场景

```
/nf-content-generator scene --outline <大纲> --setting <背景>
```

**执行方案**:

| 步骤 | 操作                                | 产出     |
| ---- | ----------------------------------- | -------- |
| 1    | 确定场景类型（室内/室外/战斗/日常） | 场景类型 |
| 2    | 根据背景设定生成场景细节            | 场景布置 |
| 3    | 添加道具、氛围元素                  | 场景道具 |
| 4    | 规划场景与情节的对应关系            | 场景节奏 |

**场景要素**:

```
场景: 宗门大殿
     │
     ├──→ 物理要素
     │      - 建筑: 大殿、柱子、匾额
     │      - 光线: 昏暗/明亮
     │      - 空间: 宽敞/狭窄
     │
     ├──→ 氛围要素
     │      - 气味: 檀香/血腥
     │      - 声音: 钟鸣/寂静
     │      - 温度: 阴冷/炎热
     │
     └──→ 情节要素
            - 关键道具: 令牌/书信
            - 关键位置: 主席/侧席
```

### dialogue - 生成对话

```
/nf-content-generator dialogue --context <上下文> --characters <角色>
```

**执行方案**:

| 步骤 | 操作                                | 产出     |
| ---- | ----------------------------------- | -------- |
| 1    | 分析角色性格和当前情绪              | 角色状态 |
| 2    | 确定对话目的（试探/威胁/求情/隐瞒） | 对话目的 |
| 3    | 生成符合角色的台词                  | 台词内容 |
| 4    | 添加潜台词和言外之意                | 潜台词   |

**对话生成方法**:

```
输入: 主角(傲娇) vs 反派(阴险)
     │
     ├──→ Step 1: 角色分析
     │      - 主角: 外冷内热、不服输
     │      - 反派: 城府深、喜欢算计
     │
     ├──→ Step 2: 对话目的
     │      - 反派: 试探主角实力
     │      - 主角: 隐藏实力、麻痹对方
     │
     ├──→ Step 3: 台词生成
     │      - 反派: "听闻少侠天赋异禀，今日一见，果然名不虚传"
     │      - 主角: "哪里哪里，在下不过是运气好罢了" (隐藏实力)
     │
     └──→ Step 4: 潜台词
            - 反派: (你在装傻，我会查清楚的)
            - 主角: (你想套我的话，没门)
```

### action - 生成动作

```
/nf-content-generator action --event <事件> --intensity <强度>
```

**执行方案**:

| 步骤 | 操作               | 产出     |
| ---- | ------------------ | -------- |
| 1    | 确定事件类型和强度 | 事件级别 |
| 2    | 规划动作序列       | 动作列表 |
| 3    | 添加战术/策略要素  | 战术设计 |
| 4    | 生成动作描写       | 动作片段 |

**战斗强度分级**:

| 强度 | 描写密度 | 动作数量    | 时长 |
| ---- | -------- | ----------- | ---- |
| 1    | 低       | 3-5个动作   | 1段  |
| 2    | 中       | 5-10个动作  | 2段  |
| 3    | 高       | 10-20个动作 | 3段+ |
| 4    | 决战     | 20+动作     | 全章 |

**动作描写模板**:

```
【低强度】 → 结果+简单动作
    "李尘一剑刺出，张伟躲避不及，被剑气所伤。"

【中强度】 → 攻防转换+多个动作
    "张伟侧身躲过剑锋，反手一掌拍向李尘胸口。
     李尘收剑格挡，却被震得后退三步。"

【高强度】 → 招式名+动作细节+环境互动
    "李尘双目圆睁，体内灵力狂涌。
     '破天三式——'
     剑光暴涨，化作三道银虹，分别刺向张伟三处要害。
     张伟面色剧变，连连后退..."
```

### emotion - 生成情感

```
/nf-content-generator emotion --character <角色> --context <上下文>
```

**执行方案**:

| 步骤 | 操作               | 产出     |
| ---- | ------------------ | -------- |
| 1    | 确定角色当前情绪   | 情绪类型 |
| 2    | 分析情绪来源和强度 | 情绪数据 |
| 3    | 用身体反应表现情绪 | 生理表现 |
| 4    | 生成内心独白       | 心理活动 |

**情绪表现方法**:

| 情绪 | 身体反应           | 内心独白           |
| ---- | ------------------ | ------------------ |
| 愤怒 | 握拳、咬牙、脸红   | "我要让你付出代价" |
| 悲伤 | 垂眼、叹息、颤抖   | "为什么偏偏是我"   |
| 喜悦 | 嘴角上扬、眼中有光 | "终于等到这一天"   |
| 恐惧 | 后退、冷汗、僵硬   | "这不可能..."      |
| 惊讶 | 瞪大眼、倒吸凉气   | "竟然是这样"       |

**情感渲染技巧**:

```
1. 用身体反应代替情绪词
   ✗ "他非常愤怒"
   ✓ "他握紧拳头，指节发白"

2. 用环境映衬情绪
   ✓ "窗外雷声轰鸣，映衬着他此刻的心情"

3. 用内心独白揭示深层情绪
   ✓ "（表面上平静）内心：她怎么能这样对我..."
```

## 错误处理

| 错误码 | 说明     | 处理方式   |
| ------ | -------- | ---------- |
| CG-001 | 大纲为空 | 自动生成   |
| CG-002 | 卷号无效 | 使用第一卷 |
| CG-003 | 章节超限 | 自动扩容   |
| CG-004 | 衔接失败 | 重新生成   |
