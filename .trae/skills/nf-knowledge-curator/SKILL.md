# 知识库管理员技能

## 系统定位

```
用户 → nf-novel-flow（主控） → nf-knowledge-curator（知识管理）
                              ├──→ /nf learn  收集知识
                              └──→ /nf query  查询知识
```

**核心职责**：

| 职责     | 说明                             |
| -------- | -------------------------------- |
| 知识沉淀 | 将创作经验转化为可复用的知识资产 |
| 知识组织 | 分类、标签、索引，便于检索       |
| 知识应用 | 为创作提供即时的知识支持         |
| 知识演进 | 持续优化、更新、淘汰过时的知识   |

## 调用方式

通过主控技能调用：

```
/nf learn --collect --from 我的作品   # 收集知识
/nf query 战斗描写技巧                # 查询知识
/nf query --type pattern --limit 5    # 查询写作模式
```

直接调用（不推荐）：

```
/nf-knowledge-curator <command> [options]
```

## 命令详解

### full - 一键完整知识管理

**功能**：自动执行 收集→整理→索引 的完整流程

```
/nf-knowledge-curator full --source <来源> [--category <分类>]
```

| 参数     | 类型   | 必填 | 默认值 | 说明         |
| -------- | ------ | ---- | ------ | ------------ |
| source   | string | 是   | -      | 知识来源路径 |
| category | string | 否   | auto   | 知识分类     |

**执行流程**：

```
┌────────────────────────────────────────────────────────────────────────┐
│                        full 命令执行流程                                 │
├────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  Step 1: collect (收集)                                                │
│  ├── 自动识别知识类型                                                   │
│  ├── 提取关键信息                                                       │
│  └── 生成原始条目                                                       │
│                                                                         │
│  Step 2: organize (整理)                                               │
│  ├── 分类归类                                                           │
│  ├── 添加标签                                                           │
│  └── 关联已有知识                                                       │
│                                                                         │
│  Step 3: index (索引)                                                  │
│  ├── 关键词提取                                                         │
│  ├── 建立索引                                                           │
│  └── 更新知识库                                                         │
│                                                                         │
└────────────────────────────────────────────────────────────────────────┘
```

**示例**：

```bash
/nf-knowledge-curator full --source ./projects/逆天剑尊 --category 内容生产
/nf learn --collect --from 逆天剑尊
```

### collect - 收集知识条目

**功能**：从项目中收集知识条目

```
/nf-knowledge-curator collect --source <来源> --type <类型> [--tag <标签>]
```

| 参数   | 类型   | 必填 | 默认值 | 说明     |
| ------ | ------ | ---- | ------ | -------- |
| source | string | 是   | -      | 来源路径 |
| type   | string | 是   | -      | 知识类型 |
| tag    | string | 否   | -      | 标签     |

**知识类型详解**：

| 类型          | 说明     | 收集来源示例                  |
| ------------- | -------- | ----------------------------- |
| lesson        | 经验教训 | 创作中发现的有效方法/失败教训 |
| pattern       | 写作模式 | 反复出现的经典场景写法        |
| template      | 创作模板 | 可直接复用的结构模板          |
| technique     | 写作技巧 | 特定写作手法                  |
| best-practice | 最佳实践 | 经过验证的有效方法            |
| case-study    | 案例研究 | 整本小说的创作复盘            |

**自动收集规则**：

```
┌────────────────────────────────────────────────────────────────────────┐
│                        自动收集识别规则                                  │
├────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  检测到以下关键词时，自动标记类型：                                       │
│                                                                         │
│  ├── "我发现..." / "成功的做法..."  → lesson (成功经验)                │
│  ├── "犯了错误..." / "踩坑..."      → lesson (失败教训)                │
│  ├── "经典开头..." / "每次都这样写" → pattern (写作模式)                 │
│  ├── "可以直接用..." / "模板如下"  → template (模板)                    │
│  └── "技巧在于..." / "关键方法是"  → technique (技巧)                   │
│                                                                         │
└────────────────────────────────────────────────────────────────────────┘
```

**示例**：

```bash
/nf-knowledge-curator collect --source ./chapters/ch-015.md --type lesson --tag 战斗
/nf learn --collect --from 逆天剑尊 --type pattern
```

### organize - 整理知识库

**功能**：对知识库进行整理、归类、重建索引

```
/nf-knowledge-curator organize [--category <分类>] [--tag <标签>] [--rebuild]
```

| 参数     | 类型   | 必填 | 默认值 | 说明     |
| -------- | ------ | ---- | ------ | -------- |
| category | string | 否   | all    | 整理分类 |
| tag      | string | 否   | -      | 标签过滤 |
| rebuild  | flag   | 否   | false  | 重建索引 |

**整理操作**：

| 操作   | 说明               |
| ------ | ------------------ |
| 归类   | 将条目放入正确分类 |
| 打标签 | 添加/修正标签      |
| 去重   | 合并重复条目       |
| 关联   | 建立条目间关联     |
| 清理   | 标记过时条目       |

**示例**：

```bash
/nf-knowledge-curator organize --category 内容生产
/nf-knowledge-curator organize --rebuild
```

### search - 搜索知识库

**功能**：搜索知识库内容

```
/nf-knowledge-curator search --query <查询> [--type <类型>] [--tag <标签>] [--limit <数量>]
```

| 参数  | 类型   | 必填 | 默认值 | 说明       |
| ----- | ------ | ---- | ------ | ---------- |
| query | string | 是   | -      | 搜索关键词 |
| type  | string | 否   | all    | 知识类型   |
| tag   | string | 否   | -      | 标签过滤   |
| limit | number | 否   | 5      | 返回数量   |

**搜索优先级**：

| 优先级 | 搜索范围       | 权重 |
| ------ | -------------- | ---- |
| 1      | 标题精确匹配   | 100  |
| 2      | 标签匹配       | 80   |
| 3      | 内容关键词匹配 | 60   |
| 4      | 相关标签扩散   | 40   |
| 5      | 分类内搜索     | 20   |

**输出格式**：

```
[1] 战斗场景模板 (BATTLE-TPL-001)
    标签: #战斗 #玄幻 #模板
    摘要: 高强度对决的标准模板，包含开局/攻防/高潮/结束结构
    适用: 主角vs强敌、越级战斗
    相关: [战斗描写技巧] [情绪引爆法]

[2] 战斗描写技巧 (TECH-002)
    标签: #战斗 #技巧 #高级
    摘要: 如何让战斗场景更紧凑更热血
    适用: 战斗章节优化
    相关: [战斗场景模板] [辰东战斗风格]
```

### template - 创建模板

**功能**：创建可复用的创作模板

```
/nf-knowledge-curator template --type <模板类型> [--name <名称>] [--based-on <参考>]
```

| 参数     | 类型   | 必填 | 默认值 | 说明       |
| -------- | ------ | ---- | ------ | ---------- |
| type     | string | 是   | -      | 模板类型   |
| name     | string | 否   | 自动   | 模板名称   |
| based-on | string | 否   | -      | 参考模板ID |

**模板类型**：

| 类型      | 说明         | 模板结构                 |
| --------- | ------------ | ------------------------ |
| chapter   | 章节模板     | 开局/发展/高潮/结尾      |
| battle    | 战斗场景模板 | 起手/攻防/绝招/结束      |
| dialogue  | 对话场景模板 | 表层/潜层/转折/结果      |
| character | 人物设定模板 | 基础/背景/性格/关系/成长 |
| world     | 世界观模板   | 世界/势力/规则/历史      |
| volume    | 卷结构模板   | 开局/铺陈/高潮/收尾      |
| plot      | 情节单元模板 | 起因/发展/转折/解决      |

**模板继承机制**：

```
模板 v2 = 模板 v1 (based-on) + 自定义修改
  ├── 保留父模板结构
  ├── 修改特定部分
  └── 添加新变体
```

**示例**：

```bash
/nf-knowledge-curator template --type battle --name 我的战斗模板
/nf-knowledge-curator template --type dialogue --based-on DIALOG-TPL-001
```

### summarize - 总结项目经验

**功能**：对项目进行复盘，生成经验总结

```
/nf-knowledge-curator summarize --project <项目> [--depth <深度>] [--output <格式>]
```

| 参数    | 类型   | 必填 | 默认值 | 说明     |
| ------- | ------ | ---- | ------ | -------- |
| project | string | 是   | -      | 项目名称 |
| depth   | string | 否   | brief  | 总结深度 |
| output  | string | 否   | md     | 输出格式 |

**深度级别**：

| 级别     | 说明     | 生成内容                     |
| -------- | -------- | ---------------------------- |
| brief    | 简要总结 | 关键指标、成功失败要点       |
| detailed | 详细复盘 | 全部分析、经验教训、改进计划 |
| complete | 完整报告 | 含案例、引用、数据支撑       |

**总结报告结构**：

```
┌────────────────────────────────────────────────────────────────────────┐
│                        项目复盘报告结构                                 │
├────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 项目概览                                                           │
│     ├── 基本信息 (名称、类型、字数、周期)                               │
│     ├── 创作数据 (章节数、完结状态、发布情况)                           │
│     └── 质量评分 (结构、人物、节奏、文笔评分)                           │
│                                                                         │
│  2. 成功经验                                                           │
│     ├── 有效的写作技巧                                                 │
│     ├── 成功的结构设计                                                 │
│     └── 验证有效的套路                                                 │
│                                                                         │
│  3. 失败教训                                                           │
│     ├── 情节漏洞                                                       │
│     ├── 人物问题                                                       │
│     └── 节奏失衡                                                       │
│                                                                         │
│  4. 改进建议                                                           │
│     ├── 针对本次项目                                                   │
│     └── 对后续项目的建议                                               │
│                                                                         │
│  5. 知识沉淀                                                           │
│     └── 可复用的模板/技巧/模式                                         │
│                                                                         │
└────────────────────────────────────────────────────────────────────────┘
```

## 知识质量标准

### 质量评估维度

| 维度   | 说明               | 权重 |
| ------ | ------------------ | ---- |
| 完整性 | 内容是否完整无缺   | 20%  |
| 准确性 | 内容是否正确可靠   | 25%  |
| 实用性 | 能否直接应用于创作 | 30%  |
| 可读性 | 表达是否清晰易懂   | 15%  |
| 时效性 | 内容是否过时       | 10%  |

### 质量评分标准

| 评分 | 等级 | 说明                 |
| ---- | ---- | -------------------- |
| 5    | 优秀 | 可直接复用，效果出色 |
| 4    | 良好 | 可用，需要少量调整   |
| 3    | 中等 | 可用，需要较大调整   |
| 2    | 较差 | 建议重构             |
| 1    | 差   | 建议淘汰             |

### 质量检查清单

```
□ 标题清晰明确
□ 内容结构完整
□ 包含实际示例
□ 标注适用场景
□ 标注不适用场景
□ 关联知识已链接
□ 无明显错误
□ 时效性在可接受范围
```

```
│                                                                         │
│  创建 → 验证 → 应用 → 优化 → 归档/淘汰                                  │
│    │       │       │       │                                           │
│    │       │       │       └── 优化后评分提升，重新进入应用             │
│    │       │       │                                                   │
│    │       │       └── 定期检查效果，收集反馈                           │
│    │       │                                                           │
│    │       └── 通过实践验证，确认有效                                   │
│    │                                                               │
│    └── 新条目标记为 pending                                           │
│                                                                         │
└────────────────────────────────────────────────────────────────────────┘
```

### 状态转换规则

| 当前状态 | 触发条件         | 下一状态 |
| -------- | ---------------- | -------- |
| pending  | 创建完成         | verified |
| pending  | 需要验证         | testing  |
| testing  | 验证通过         | verified |
| testing  | 验证失败         | archived |
| verified | 评分≥4，长期有效 | verified |
| verified | 评分<3，或过时   | archived |
| archived | 可重新验证       | testing  |
| archived | 无价值           | deleted  |

## 错误处理

| 错误码 | 说明         | 处理方式                 |
| ------ | ------------ | ------------------------ |
| KC-001 | 来源不存在   | 提示检查路径             |
| KC-002 | 知识类型错误 | 显示可用类型列表         |
| KC-003 | 整理失败     | 回滚本次修改，保留原数据 |
| KC-004 | 搜索无结果   | 建议扩大搜索范围         |
| KC-005 | 模板创建失败 | 检查必填字段             |
| KC-006 | 导出失败     | 检查格式和内容           |
| KC-007 | 权限不足     | 检查文件读写权限         |
| KC-008 | 知识库损坏   | 使用备份恢复             |

**错误恢复流程**：

```
KC-001: 检查 --source 路径是否正确
      → 修正路径后重新执行

KC-002: 显示所有可用知识类型
      → 根据提示重新执行

KC-004: 扩大搜索范围
      → 使用更模糊的关键词

KC-008: 执行知识库恢复
      → /nf-knowledge-curator restore --backup <latest>
```
