# 专业优化技能

## 使用场景

- 用户要优化章节内容
- 用户觉得文章有AI味道
- 用户要优化对话
- 用户要优化战斗描写
- 用户要优化卷间衔接

---

## 命令

### 生成优化章节正文

> 优化章节内容，去AI味道、统一对话、战斗描写、卷间衔接。

```
/nf-specialist-optimizer
```

**执行流程**:

1. style (文风优化)：去AI味道、统一文风
2. plot (情节优化)：逻辑矛盾、节奏调整、爽点密度
3. dialogue (对话优化)：口语化、潜台词、人物性格
4. battle (战斗优化)：动作紧凑、战术清晰
5. connection (衔接优化)：卷间连贯性
6. 输出优化后的章节正文

## 流程

### style - 文风优化

**执行流程**:

```
原文 → 识别AI特征 → 针对性优化 → 文风统一稿
```

**AI特征识别与优化**:

| AI特征     | 识别模式                 | 优化方法                   |
| ---------- | ------------------------ | -------------------------- |
| 过度解释   | "他很生气，因为他被骗了" | 删除解释，保留"他握紧拳头" |
| 机械化副词 | "缓缓地"、"轻轻地"       | 删除或替换                 |
| 重复句式   | "......"                 | 变化句式长度               |
| 情绪词堆砌 | "非常愤怒、极度生气"     | 用身体反应替代             |
| 对话直白   | "我很生气！"             | 增加潜台词                 |

**AI味道检测表**:

| 检测项目   | 特征                 | 示例                                 | 检测规则       |
| ---------- | -------------------- | ------------------------------------ | -------------- |
| 过度完美   | 形容词堆砌           | "深邃的棕色眼眸...智慧与神秘"        | 连续形容词>2   |
| 因果完整   | 所有事件有完整因果链 | "因为A...于是C...导致D...最后E"      | 连续因果词>2   |
| 过度解释   | 作者旁白过多         | "这让他感到非常愤怒，毕竟谁都会生气" | 连续解释句>1   |
| 情感直白   | 直接陈述情感         | "她感到非常开心和幸福"               | 情感词+"感到"  |
| 格式化对仗 | 句式过于整齐         | "日出东方，唯我独尊；脚踏实地..."    | 结构相似>2     |
| 无意义填充 | 为凑字数的描写       | "阳光明媚，天空湛蓝，白云飘飘"       | 无信息形容词>3 |

**去AI味道技巧**:

| 技巧            | 说明                   | 练习                     |
| --------------- | ---------------------- | ------------------------ |
| Show Don't Tell | 用动作展示，不直接陈述 | 将"感到XX"替换为身体反应 |
| 打断完美感      | 故意引入不完美         | 在完美描述后加转折       |
| 留白艺术        | 不解释，让读者自己理解 | 删除50%解释句            |
| 声音节奏        | 变化句式长度           | 将30%长句改为短句        |

**去AI味道执行方案**:

| 步骤 | 操作               | 产出     |
| ---- | ------------------ | -------- |
| 1    | 识别原文中的AI特征 | 问题列表 |
| 2    | 删除过度解释性词语 | 简洁表达 |
| 3    | 替换机械化副词     | 自然表达 |
| 4    | 情绪用身体反应表现 | 具象化   |
| 5    | 对话口语化         | 自然对话 |
| 6    | 长短句交错         | 节奏感   |

**示例**:

```
【原文(AI味)】
他缓缓地睁开眼睛，心中充满了惊讶。他轻轻地坐起身来，回忆着刚才发生的一切。

【优化后(去AI味)】
他睁开眼。
记忆还停留在那一剑刺来的瞬间。
```

### plot - 情节优化

**执行流程**:

```
原文 → 逻辑检查 → 节奏分析 → 爽点评估 → 优化稿
```

**逻辑矛盾检查**:

| 检查项   | 示例                         | 解决方案       |
| -------- | ---------------------------- | -------------- |
| 战力崩坏 | 前章被秒杀，后章秒杀对方     | 调整实力对比   |
| 时间矛盾 | A在X地说完话，B立刻在Y地回应 | 增加过渡或调整 |
| 物品矛盾 | 前面说丢失，后面突然出现     | 删除或解释来源 |
| 性格矛盾 | 人物行为与性格设定不符       | 修正行为       |
| 常识矛盾 | 物理/逻辑违反设定            | 调整描写       |

**节奏调整方法**:

| 问题     | 表现                   | 解决方案           |
| -------- | ---------------------- | ------------------ |
| 节奏过慢 | 大段描写、内心独白过多 | 删除冗余，增加冲突 |
| 节奏过快 | 情节跳跃、缺乏过渡     | 增加铺垫，丰富细节 |
| 高潮不足 | 平铺直叙               | 增加反转或爽点     |

**爽点密度评估**:

```
章节爽点分布:
  - 章初: 期待感铺垫
  - 章中: 小爽点1-2个
  - 章中后: 中爽点1个
  - 章末: 大爽点/悬念

优化目标: 每3000字至少1个有效爽点
```

### dialogue - 对话优化

**执行流程**:

```
原文对话 → 人物性格分析 → 潜台词设计 → 节奏调整 → 优化稿
```

**对话优化方案**:

| 步骤 | 操作                         | 产出       |
| ---- | ---------------------------- | ---------- |
| 1    | 提取原文所有对话             | 对话列表   |
| 2    | 分析每句对话的角色性格符合度 | 问题点     |
| 3    | 设计潜台词和言外之意         | 潜台词设计 |
| 4    | 调整对话节奏（长短句搭配）   | 节奏优化   |
| 5    | 替换不符合性格的表达         | 性格统一   |

**对话优化标准方法论**:

| 方法                   | 操作                        | 内容                |
| ---------------------- | --------------------------- | ------------------- |
| **D1: 人物性格映射法** | 建立角色性格-语言风格映射表 | 性格类型→语言特征   |
| **D2: 潜台词设计法**   | 为关键对话设计言外之意      | 表面+暗示+深意      |
| **D3: 对话节奏控制法** | 通过句式长短、密度控制节奏  | 紧张/舒缓/冲突/悬疑 |
| **D4: 对话标签多样化** | 避免单一使用"说"            | 动作+语气+方式      |

**D1: 人物性格映射法**:

| 性格类型 | 语言特征                       |
| -------- | ------------------------------ |
| 腹黑型   | 表面客气、话里有刺、点到即止   |
| 阳光型   | 直接热情、感叹词多、语气上扬   |
| 冷漠型   | 短句为主、少有感叹、不主动开口 |
| 傲娇型   | 嘴硬心软、口是心非、别扭表达   |
| 霸气型   | 简短有力、命令式、不容置疑     |
| 腹诽型   | 内心吐槽丰富、表面一本正经     |

**D2: 潜台词设计法**:

| 层次         | 说明                     | 示例             |
| ------------ | ------------------------ | ---------------- |
| 第一层(表面) | 直接说出的内容           | "你做得很好"     |
| 第二层(暗示) | 通过语气、停顿暗示的态度 | (冷笑)           |
| 第三层(深意) | 需要读者解读的真正意图   | 讽刺对方做得很差 |

**D3: 对话节奏控制法**:

| 节奏类型 | 特征                           | 适用场景  |
| -------- | ------------------------------ | --------- |
| 紧张节奏 | 短句连发、问答紧凑、无喘息     | 危机/战斗 |
| 舒缓节奏 | 长短交替、有描写间隔、氛围轻松 | 日常/温馨 |
| 冲突节奏 | 互相打断、语速加快、情绪升级   | 争吵/对峙 |
| 悬疑节奏 | 一问一答、留白、意味深长       | 推理/试探 |

**D4: 对话标签多样化**:

| 标签类型 | 示例                            |
| -------- | ------------------------------- |
| 语气词   | 哼/嗯/啊/哦/呀/嘛/啦            |
| 动作标签 | 笑/怒/叹/愣/皱眉/挑眉/摇头/点头 |
| 方式词   | 低声道/大声道/压低声音/一字一顿 |
| 情绪词   | 嘲讽/真诚/戏谑/冷淡/温柔/冰冷   |

**人物性格对话模板库**:

| 性格类型 | 常用词汇       | 句式特征                   | 语气特征       | 禁忌词         |
| -------- | -------------- | -------------------------- | -------------- | -------------- |
| 腹黑型   | 当然/的确/不过 | 反问句、设问句             | 温和中带刺     | 太直白         |
| 阳光型   | 太好了/加油    | 感叹句、完整句             | 热情、上扬     | 冷漠词         |
| 冷漠型   | 嗯/哦/随便     | 短句、省略句               | 平淡、无起伏   | 感叹词         |
| 傲娇型   | 才不是/谁稀罕  | 否定句、口是心非           | 别扭、不自然   | 太坦诚         |
| 霸气型   | 听令/必须/给我 | 命令句、祈使句             | 强硬、不容置疑 | 疑问、商量的   |
| 腹诽型   | 表面正常       | 内心OS丰富、与说出口的相反 | 表里不一       | 内心和表面一致 |

**对话优化示例**:

```
【原文】
"你好厉害啊！"小美说。
"没什么，只是运气好。"主角回答。

【优化后】
小美眼睛亮晶晶的："你这也太强了吧！"
主角耸耸肩，嘴角带着一丝不易察觉的笑意："运气好而已。"
小美凑近了些："骗谁呢，你当我眼瞎？"

【腹黑型优化】
小美："做得真不错呢~"
主角："承蒙夸奖。"(内心: 就这水平？)
小美："不过下次别再让我失望了哦。"(话里有话)

【傲娇型优化】
小美："哼，我才不觉得你厉害呢！"
主角："哦？那刚才喊加油的是谁？"
小美："那是...那是别人让我喊的！"
```

**对话优化规则库**:

| 规则ID     | 名称         | 描述                 | 检测            | 优化方法           |
| ---------- | ------------ | -------------------- | --------------- | ------------------ |
| DIALOG-001 | 省略主语     | 口语对话省略主语     | "你XX" / "我XX" | 口语化省略         |
| DIALOG-002 | 口语化       | 使用口语词汇         | 书面语→口语     | 替换为口语词       |
| DIALOG-003 | 标签多样化   | 避免只用"说"         | 100%用"说"      | 增加动作/表情/语气 |
| DIALOG-004 | 潜台词       | 重要对话应有言外之意 | 直白>70%        | 增加暗示层次       |
| DIALOG-005 | 身份符合     | 对话符合角色背景     | 博士说"哇塞"    | 调整为符合身份     |
| DIALOG-006 | 情绪连贯     | 对话情绪需连贯       | 哭→笑→哭        | 调整过渡           |
| DIALOG-007 | 问句功能     | 问句需有实际功能     | 废话问句        | 删除或合并         |
| DIALOG-008 | 对话密度控制 | 对话不能过密或过稀   | 连续10句无描写  | 增加动作/描写      |

**潜台词设计标准**:

| 场景类型 | 表面台词       | 潜台词设计     | 暗示方式      |
| -------- | -------------- | -------------- | ------------- |
| 讽刺     | "你真厉害"     | 实际在贬低     | 表情冷淡/冷笑 |
| 试探     | "你最近怎么样" | 实际在打探情报 | 眼神闪烁      |
| 关心     | "别太累了"     | 实际很担心     | 欲言又止      |
| 拒绝     | "我再想想"     | 实际是拒绝     | 拖延/回避     |
| 挑衅     | "就凭你？"     | 实际看不起对方 | 轻蔑语气      |
| 告白     | "其实你人不错" | 实际想说喜欢   | 欲言又止/脸红 |

### battle - 战斗优化

**执行流程**:

```
战斗段落 → 强度评估 → 动作分析 → 战术设计 → 紧张感渲染 → 优化稿
```

**战斗优化方案**:

| 步骤 | 操作                 | 产出     |
| ---- | -------------------- | -------- |
| 1    | 提取战斗段落         | 战斗片段 |
| 2    | 评估战斗强度等级     | 强度定级 |
| 3    | 分析动作描写的精准度 | 动作问题 |
| 4    | 设计战术和策略       | 战术优化 |
| 5    | 渲染紧张氛围         | 氛围增强 |

**战斗描写标准方法论**:

| 方法                 | 说明                         |
| -------------------- | ---------------------------- |
| **B1: 战斗强度分级** | 4级强度分类及字数标准        |
| **B2: 动作描写标准** | 精准度、避免重复、动作连贯   |
| **B3: 节奏控制**     | 短句、长短句交替、留白       |
| **B4: 氛围渲染**     | 环境/心理/感官/节奏变化      |
| **B5: 逻辑一致性**   | 战力体系、能力限制、伤势影响 |

**B1: 战斗强度分级**:

| 等级 | 类型                 | 结构                     | 要素                | 字数       |
| ---- | -------------------- | ------------------------ | ------------------- | ---------- |
| 1级  | 低强度:秒杀/碾压     | 结果→2-3动作→结束        | 一招制胜快速结束    | 100-200字  |
| 2级  | 中强度:有来有回      | 试探→攻防2-3轮→结果      | 互相试探有来有回    | 300-500字  |
| 3级  | 高强度:苦战/技能展示 | 开局→苦战→底牌→反转      | 招式名/10+动作/细节 | 800-1500字 |
| 4级  | 决战级:生死战/逆转   | 多回合→心理→环境→20+动作 | 生死战/逆转/悬念    | 2000+字    |

**B2: 战斗动作描写标准**:

| 标准     | 要求                                                                            | 示例                                      |
| -------- | ------------------------------------------------------------------------------- | ----------------------------------------- |
| 精准度   | 武器(刺/劈/砍/挑/扫/挡/格)·身体(闪/躲/退/进/跃/扑/滚)·灵力(凝聚/释放/冲击/防御) | -                                         |
| 避免重复 | 同一动词最多连续2次                                                             | 避免:打出打出打出 → 正确:挥拳、侧踢、下劈 |
| 动作连贯 | 每个动作后需有结果或下一个动作                                                  | A动作→结果/B动作→结果                     |

**B3: 战斗节奏控制**:

| 技巧       | 用途             | 示例                                                         |
| ---------- | ---------------- | ------------------------------------------------------------ |
| 短句       | 关键一击、胜负手 | 剑落。血溅。人倒。                                           |
| 长短句交替 | 描写攻防转换     | 对方攻势如潮，他连连后退。但他没慌。就在对方露出破绽的瞬间—— |
| 留白       | 让读者想象       | 那一剑...无法形容。                                          |

**B4: 战斗氛围渲染**:

| 要素     | 说明                 |
| -------- | -------------------- |
| 环境描写 | 1-2句环境变化/破坏   |
| 心理描写 | 关键战斗需有内心活动 |
| 感官描写 | 声音、光影、气息     |
| 节奏变化 | 快→慢→快的节奏循环   |

**B5: 战斗逻辑一致性**:

| 要素     | 说明                      |
| -------- | ------------------------- |
| 战力体系 | 严格遵守设定中的等级/实力 |
| 能力限制 | 技能有冷却/消耗/副作用    |
| 伤势影响 | 受伤后实力下降            |
| 环境影响 | 地形/天气对战斗的影响     |

**战斗描写模板**:

| 强度      | 结构                       | 要素              |
| --------- | -------------------------- | ----------------- |
| 低(1级)   | 结果+2-3动作               | 一招制胜/快速结束 |
| 中(2级)   | 攻防转换+5-8动作           | 有来有回/试探     |
| 高(3级)   | 招式名+动作细节+10+动作    | 苦战/技能展示     |
| 决战(4级) | 多个回合+心理+环境+20+动作 | 生死战/逆转       |

**战斗描写标准流程**:

| 步骤   | 操作         | 内容                                                          |
| ------ | ------------ | ------------------------------------------------------------- |
| Step 1 | 战斗片段提取 | 识别开始标志(交手/开战/出手)、结束标志(胜负已分/倒下/逃离)    |
| Step 2 | 强度评估     | 检查战斗时长、动作数量、底牌/逆转、判定强度等级               |
| Step 3 | 动作分析     | 统计动作类型(武器/身体/灵力)、检查重复度/精准度、列出优化清单 |
| Step 4 | 战术设计     | 分析双方实力对比、设计攻防转换/底牌逆转点、补充战术逻辑       |
| Step 5 | 氛围增强     | 插入环境变化、补充心理活动、添加感官细节、调整节奏            |
| Step 6 | 输出优化稿   | -                                                             |

**战斗描写规则库**:

| 规则ID     | 名称         | 描述                      | 正例                             |
| ---------- | ------------ | ------------------------- | -------------------------------- |
| BATTLE-001 | 动作连续限制 | 不超过3个连续短句         | "他出拳。对方侧身躲过，反手一肘" |
| BATTLE-002 | 插入环境描写 | 每2-3动作插入1句环境/心理 | "剑气斩落。地面炸裂，碎石四溅"   |
| BATTLE-003 | 避免动作重复 | 同一动作最多出现2次       | "打"→["挥出","劈出","斩下"]      |
| BATTLE-004 | 力量等级分明 | 不同等级应有明显区别      | 弱方防守多，强方一招制敌         |
| BATTLE-005 | 感受真实化   | 允许肌肉酸痛/呼吸急促     | 避免"感受到能量在体内游走"       |
| BATTLE-006 | 招式名使用   | 高强度战斗应有招式名      | "破天三式——第二式"               |
| BATTLE-007 | 伤势体现     | 战斗应有伤亡/消耗体现     | "右臂发麻，灵力消耗过半"         |
| BATTLE-008 | 胜负逻辑     | 胜负需有合理原因          | 不能突然实力暴涨/暴跌            |

### connection - 卷间衔接

**执行流程**:

```
上卷结尾 → 衔接分析 → 情节连贯 → 人物状态 → 文风统一 → 优化稿
```

**衔接优化方案**:

| 步骤 | 操作                   | 产出       |
| ---- | ---------------------- | ---------- |
| 1    | 分析上卷结尾的关键信息 | 承接要点   |
| 2    | 检查下卷开头的承接     | 连贯性评估 |
| 3    | 优化情节衔接           | 流畅过渡   |
| 4    | 统一人物状态           | 状态一致   |
| 5    | 统一文风               | 文风连贯   |

**衔接检查清单**:

```
□ 人物状态: 伤势、能力、情绪是否延续
□ 物品道具: 重要物品是否携带/保管
□ 剧情进度: 悬念是否解答/延续
□ 时间线: 间隔多久、是否合理
□ 势力关系: 各方态度是否有变
□ 世界观: 规则/设定是否一致
```

**卷间衔接优化标准流程 (3C标准)**:

| 阶段                   | 类别         | 要点                                                                                         |
| ---------------------- | ------------ | -------------------------------------------------------------------------------------------- |
| **C1: 承接要素提取**   | 必接要素     | 人物状态(伤势/能力/情绪)、物品道具(关键物品位置)、剧情悬念(待解答的问题)、时间节点(间隔多久) |
|                        | 次要要素     | 势力关系(各方态度变化)、世界观(规则/设定延续)、伏笔(跨卷伏笔检查)                            |
| **C2: 衔接类型判定**   | 时间跳跃衔接 | 需明确标注时间流逝、简述期间发生的重要事件、更新人物状态变化原因                             |
|                        | 事件连续衔接 | 无需大段时间过渡、直接承接上卷结尾、保持叙事节奏                                             |
|                        | 视角转换衔接 | 新视角需有简短背景回顾、避免新读者懵                                                         |
| **C3: 过渡段写作标准** | 字数标准     | 300-800字                                                                                    |
|                        | 内容要求     | 承接上卷结尾、简述时间间隔内变化、引入新卷核心冲突、留下新悬念                               |
|                        | 禁忌         | 大段回忆(让读者出戏)、信息过载(一股脑倒出)、与前文矛盾                                       |
