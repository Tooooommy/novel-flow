## 一、节奏量化体系

### 1.1 情感曲线模型

```yaml
情感曲线:
  描述: "将文本情感量化为 -1.0 到 +1.0 的连续值"

  计算方法:
    基础值:
      极度负面: -1.0
      强烈负面: -0.7
      中度负面: -0.4
      轻微负面: -0.2
      中性: 0.0
      轻微正面: 0.2
      中度正面: 0.4
      强烈正面: 0.7
      极度正面: 1.0

    情感词权重:
      强情感词: 1.5
      中情感词: 1.0
      弱情感词: 0.5

    上下文调节:
      转折词(但是/然而): 方向反转 × 0.8
      递进词(更/尤其): 当前值 × 1.2
      对比词(而/却): 取两者差值

  章节情感目标:
    示例章节(3000字):
      开头: 0.0 ~ 0.1 (建立)
      发展: 0.1 ~ 0.3 (铺垫)
      高潮: 0.6 ~ 0.9 (爆发)
      回落: 0.3 ~ 0.1 (余韵)
```

### 1.2 爽点密度计算

```yaml
爽点密度:
  描述: "衡量单位字数内爽点的密集程度"

  爽点类型:
    逆转爽点:
      权重: 1.5
      关键词: "没想到", "竟然", "出乎意料", "反杀", "打脸"
      检测方式: "逆转型句子 + 结果反转"

    升级爽点:
      权重: 1.3
      关键词: "突破", "晋升", "觉醒", "解锁", "获得"
      检测方式: "实力/地位提升事件"

    情报爽点:
      权重: 1.2
      关键词: "原来", "真相", "秘密", "揭露"
      检测方式: "关键信息披露"

    情感爽点:
      权重: 1.2
      关键词: "心动", "感动", "热血", "落泪"
      检测方式: "强烈情感表达"

  密度计算公式:
    公式: "S = (Σ 爽点权重) / 字数 × 1000"
    单位: "个/千字"

  目标值:
    快餐文学: "≥ 15 个/千字"
    精品网文: "8-15 个/千字"
    传统文学: "3-8 个/千字"
```

## 二、规则库定义

### 2.1 战斗描写规则库

```yaml
战斗描写规则:
  核心原则: "简洁有力，视觉化，避免过度解释"

  规则列表:
    - 规则ID: "BATTLE-001"
      名称: "动作连续限制"
      描述: "连续动作描写不超过3个短句"
      正例: |
        他出拳。对方侧身躲过，反手一肘。
      反例: |
        他快速出拳，对手敏捷地侧身躲闪，然后迅速反击，
        动作一气呵成，毫无停顿。
      检测: "连续短句数 > 3"

    - 规则ID: "BATTLE-002"
      名称: "插入环境描写"
      描述: "招式碰撞前后必须有环境或心理描写"
      频率: "每2-3个动作插入1句环境/心理"
      正例: |
        剑气斩落。地面炸裂，碎石四溅。
        他握紧剑柄，心跳如鼓。

    - 规则ID: "BATTLE-003"
      名称: "避免动作重复"
      描述: "同一动作在一段内最多出现2次"
      替代词库:
        "打": ["挥出", "劈出", "斩下", "击向"]
        "看": ["盯着", "望向", "注视着", "扫视"]

    - 规则ID: "BATTLE-004"
      名称: "力量等级分明"
      描述: "不同实力等级角色的战斗应有明显区别"
      弱对强: "防守多，躲避多，反击少"
      强对弱: "一招制敌，少量描写"
      同级对抗: "互有攻守，紧张激烈"

    - 规则ID: "BATTLE-005"
      名称: "感受真实化"
      描述: "战斗中的感受要真实"
      允许的描写:
        - 肌肉酸痛
        - 呼吸急促
        - 视线模糊
        - 疼痛传导
      避免的描写:
        - "感受到XX的能量"
        - "仿佛有XX在体内游走"
        - 抽象的能量描述
```

### 2.2 对话优化规则库

```yaml
对话优化规则:
  核心原则: "符合人物性格，推动情节，增加潜台词"

  规则列表:

    - 规则ID: "DIALOG-001"
      名称: "省略主语"
      描述: "口语对话省略主语，增加自然感"
      正例: |
        "吃饭了吗？" "还没。"
      反例: |
        "你吃饭了吗？" "我还没有吃饭。"

    - 规则ID: "DIALOG-002"
      名称: "口语化表达"
      描述: "使用口语词汇而非书面语"
      书面语 → 口语:
        "我明白了" → "哦"
        "再见了" → "拜拜"/"走了"
        "为什么不" → "咋不"
        "不知道" → "不清楚"/"没谱"

    - 规则ID: "DIALOG-003"
      名称: "对话标签多样化"
      描述: "避免只用'说'作为对话标签"
      替代词库:
        低强度: ["道", "应", "喊", "念"]
        中强度: ["问", "笑", "哼", "叹"]
        高强度: ["吼", "咆哮", "厉声道", "怒斥"]

    - 规则ID: "DIALOG-004"
      名称: "潜台词规则"
      描述: "重要对话应有言外之意"
      检测: "直白表达 > 70% 标记为'AI味道'"
      改进方法:
        - 反问代替陈述
        - 动作代替语言
        - 沉默代替回答
        - 转移话题

    - 规则ID: "DIALOG-005"
      名称: "符合人物身份"
      描述: "对话用词要符合角色背景"
      检测: "生僻词 > 5% 标记为'不符合'"
```

### 2.3 场景描写规则库

```yaml
场景描写规则:
  核心原则: "五感并用，氛围统一，详略得当"

  规则列表:
    - 规则ID: "SCENE-001"
      名称: "五感平衡"
      描述: "场景描写应包含至少2种感官"
      视觉: "颜色、形状、动态"
      听觉: "声音、节奏、静默"
      嗅觉: "气味、香臭"

      味觉: "在特定场景使用"

    - 规则ID: "SCENE-002"
      名称: "详略得当"
      描述: "核心场景详写，背景略写"
      核心场景: "50-100字/描写点"
      背景场景: "10-20字/描写点"

    - 规则ID: "SCENE-003"
      名称: "氛围统一"
      描述: "所有描写服务于整体氛围"
      快乐场景: "明亮色彩，轻快声音"
      悲伤场景: "灰暗色彩，低沉声音"
      紧张场景: "短促节奏，高频动作"

    - 规则ID: "SCENE-004"
      名称: "视角一致性"
      描述: "场景描写从当前视角人物出发"
      检测: "第三人称全知视角 > 30% 标记为'偏离'"
```

## 三、AI味道检测

### 3.1 AI特征检测表

```yaml
AI味道检测:
  描述: "检测文本中的AI生成特征"

  检测项目:

    - 项目: "过度完美的描述"
      特征: "形容词堆砌，细节过于精确"
      示例: "他有一双深邃的棕色眼眸，仿佛藏着无尽的智慧与神秘"
      检测规则: "连续形容词 > 2 个"

    - 项目: "因果链完整"
      特征: "所有事件都有完整的前因后果"
      示例: "因为A，所以B，于是C，导致D，最后E"
      检测规则: "连续因果词 > 2 个"

    - 项目: "过度解释"
      特征: "作者旁白过多，替读者总结"
      示例: "这让他感到非常愤怒，毕竟谁遇到这种事都会生气的"
      检测规则: "连续解释句 > 1 句"

    - 项目: "情感直白"
      特征: "直接陈述情感，不通过动作展示"
      示例: "她感到非常开心和幸福"
      检测规则: "情感词 + "感到" 出现"

    - 项目: "格式化对仗"
      特征: "句式过于整齐对仗"
      示例: "日出东方，唯我独尊；脚踏实地，方得始终"
      检测规则: "结构相似句 > 2 连续"

    - 项目: "无意义填充"
      特征: "为凑字数的环境描写"
      示例: "这是一个阳光明媚的早晨，天空湛蓝，白云飘飘"
      检测规则: "无信息量形容词 > 3 个"
```

    - 规则: "能力使用约束"

    - 规则: "物品归属约束"
