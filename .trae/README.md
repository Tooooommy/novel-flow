## 二、完整流程详解

### 阶段1：Think (思考与创意)

**目标**：从用户模糊想法到清晰创意方案

**智能体**：创意探索师、市场分析师、创新评估师

**工作流程**：

```
用户输入 → 创意发散 → 市场分析 → 创意评估 → 方案确定
```

**核心技能**：

#### 1. 创意发散技能

```
触发条件：当用户提出"想写小说"或模糊创意时
执行步骤：
1. 分析用户原始输入，提取关键词
2. 进行关联性联想，生成5个创意方向
3. 为每个方向添加创新元素
4. 评估每个方向的商业潜力
5. 输出结构化创意提案
```

**创意提案模板**：

```markdown
## 创意方向：[方向名称]

### 核心概念

[一句话概括故事]

### 创新亮点

1. [创新点1：与传统差异]
2. [创新点2：新颖设定]
3. [创新点3：独特视角]

### 市场匹配度

- 当前热度：★★★☆☆
- 竞争程度：中等
- 目标读者：[具体画像]
- 平台偏好：[适配平台]

### 风险评估

- 创新风险：[高/中/低]
- 执行难度：[高/中/低]
- 市场风险：[高/中/低]

### 推荐理由

[为什么选择这个方向]
```

#### 2. 市场分析技能

```
触发条件：创意生成后
执行步骤：
1. 分析目标平台趋势
2. 研究竞品表现
3. 分析读者偏好变化
4. 识别市场机会
5. 生成市场分析报告
```

**市场报告结构**：

- 平台数据概览
- 竞品案例分析
- 读者行为洞察
- 趋势预测
- 创作建议

### 阶段2：Plan (规划与设计)

**目标**：将创意转化为可执行蓝图

**智能体**：篇幅规划师、世界构建师、故事架构师、人物设计师

**工作流程**：

```
创意输入 → 篇幅确定 → 世界观构建 → 故事架构 → 人物设计 → 大纲输出
```

**核心技能**：

#### 1. 篇幅规划技能

```
触发条件：创意确定后
决策逻辑：
- 短篇（10-30万字）：适合新手、试水作品
- 中篇（30-100万字）：主流网络小说
- 长篇（100-300万字）：大神作品、系列开篇
- 超长篇（300万+）：顶级IP、世界观宏大
```

**篇幅规划表**：

```yaml
总字数: 1000000
预计章节: 300章
单章字数: 3000-3500
更新频率: 每日2章
创作周期: 150天
缓冲存稿: 30章
```

#### 2. 世界构建技能

```
构建层次：
1. 表层世界（物理环境）
2. 中层世界（社会结构）
3. 深层世界（规则体系）
4. 隐藏世界（秘密设定）
```

**世界设定文档**：

```markdown
# 世界观设定

## 1. 物理世界

- 地理环境
- 气候特征
- 资源分布
- 生物体系

## 2. 社会世界

- 政治结构
- 经济体系
- 文化传统
- 阶级分层

## 3. 超自然体系

- 力量等级
- 修炼体系
- 特殊规则
- 禁忌限制
```

#### 3. 多线故事架构技能

```
故事线类型：
- 主线：核心剧情，贯穿始终
- 暗线：隐藏真相，逐步揭示
- 支线：分支剧情，丰富世界
- 感情线：人物关系发展
- 成长线：主角能力成长
```

**故事线规划矩阵**：
| 章节段 | 主线发展 | 暗线进展 | 支线展开 | 感情推进 | 成长节点 |
|--------|----------|----------|----------|----------|----------|
| 1-30章 | 入门修行 | 身世之谜 | 结识伙伴 | 初遇女主 | 筑基成功 |
| 31-60章 | 门派大比 | 阴谋初现 | 探索秘境 | 感情升温 | 金丹突破 |
| ... | ... | ... | ... | ... | ... |

#### 4. 人物设计技能

```
人物档案要素：
1. 基本信息（姓名、年龄、外貌）
2. 性格特征（核心性格、隐藏性格）
3. 欲望恐惧（核心驱动力）
4. 成长弧线（初始→转变→终点）
5. 关系网络（重要人际关系）
```

```

    主角 --> 爱人
    主角 --> 兄弟
    主角 --> 仇敌
    师父 --> 宗门
    ...
```

### 阶段3：Build (构建与创作)

**目标**：将蓝图转化为具体内容

**智能体**：节奏工程师、内容生成师

**工作流程**：

```
大纲输入 → 节奏设计 → 章节生成 → 质量自检 → 草稿输出
```

**核心技能**：

#### 1. 节奏设计技能

```
节奏控制原则：
- 每章：开头钩子 + 中间发展 + 结尾悬念
- 每3章：一个小高潮
- 每10章：一个大转折
- 每30章：一个剧情节点
```

**章节节奏模板**：

```markdown
# 第X章节奏设计

## 字数分配

- 开头钩子：200-300字
- 情节发展：1500-2000字
- 冲突高潮：800-1000字
- 结尾悬念：300-500字
- 总计：3000字左右

## 情绪曲线

- 起始情绪：[紧张/轻松/悬疑]
- 发展情绪：[逐渐升级]
- 高潮情绪：[强烈情感]
- 收尾情绪：[留下悬念]

## 爽点布置

- 位置：[具体段落]
- 类型：[实力提升/打脸/反转等]
- 强度：[小/中/大]
```

#### 2. 章节生成技能

```
生成步骤：
1. 读取本章大纲
2. 获取相关人物设定
3. 获取场景环境描述
4. 应用节奏模板
5. 生成章节草稿
6. 自检基础质量
```

**章节草稿结构**：

```markdown
# 第X章 [章节标题]

[开头：立即抓住读者]

[发展：推进情节，展现人物]

[冲突：制造紧张，设置障碍]

[高潮：解决问题，提供爽点]

[结尾：留下悬念，引导阅读]
```

### 阶段4：Review (审查与优化)

**目标**：专业化的内容优化

**智能体**：专业优化师

**工作流程**：

```
草稿输入 → 专项优化 → 整合优化 → 优化稿输出
```

**核心技能**：

#### 1. 专项优化技能集群

```
优化模块：
1. 对白优化师
   - 检查对话是否符合人物性格
   - 优化对话节奏
   - 增加潜台词

2. 场景描述师
   - 环境描写优化
   - 氛围营造增强
   - 感官细节丰富

3. 战斗描写师
   - 动作设计
   - 战术逻辑
   - 紧张感营造

4. 情感渲染师
   - 情感表达优化
   - 心理描写增强
   - 情感转折自然化

5. 节奏微调师
   - 段落拆分/合并
   - 句子长短调整
   - 阅读流畅度优化
```

**优化检查表**：

- [ ] 对话：符合人物、推动情节
- [ ] 描写：生动具体、不冗余
- [ ] 节奏：张弛有度、不拖沓
- [ ] 情感：真实自然、有共鸣
- [ ] 逻辑：前后一致、合情理

### 阶段5：Test (测试与验证)

**目标**：质量保证与风险控制

**智能体**：逻辑审查官、安全合规官、质量评估师

**工作流程**：

```
优化稿输入 → 逻辑审查 → 安全审查 → 质量评估 → 测试报告
```

**核心技能**：

#### 1. 逻辑一致性审查技能

```
审查维度：
1. 时间线一致性
   - 时间顺序合理
   - 时间跨度恰当
   - 无时间矛盾

2. 人物行为一致性
   - 性格前后一致
   - 能力合理变化
   - 动机符合设定

3. 世界观一致性
   - 规则自洽
   - 设定不冲突
   - 发展合理

4. 情节逻辑一致性
   - 因果关系合理
   - 转折有铺垫
   - 结局必然
```

**逻辑审查报告**：

```yaml
审查时间: [时间]
审查章节: 第X章
审查结果: 通过/不通过

发现的问题:
  - 问题1: [描述]
  - 问题2: [描述]

修改建议:
  - 建议1: [具体修改]
  - 建议2: [具体修改]

严重程度: 高/中/低
需要修改: 是/否
```

#### 2. 安全合规审查技能

```
审查标准：
1. 政策合规性
   - 不涉及敏感政治
   - 不传播违法信息
   - 不宣扬暴力

2. 平台规则符合性
   - 符合平台内容规范
   - 不侵犯他人版权
   - 不恶意竞争

3. 道德伦理合规性
   - 价值观正向
   - 不传播不良信息
   - 尊重文化传统
```

**安全审查清单**：

- [ ] 无政治敏感内容
- [ ] 无违法信息
- [ ] 无过度暴力描写
- [ ] 无不良价值观
- [ ] 无版权风险
- [ ] 符合平台规则

#### 3. 内容质量评估技能

```
评估指标：
1. 可读性评分（0-10）
2. 节奏评分（0-10）
3. 爽点密度（每千字数）
4. 钩子强度（0-1）
5. 情感张力（0-1）
6. 创新性评分（0-10）
```

**质量评估报告**：

```markdown
# 第X章质量评估

## 量化指标

- 可读性: 8.2/10
- 节奏: 7.8/10
- 爽点密度: 0.4（每千字）
- 钩子强度: 0.85
- 情感张力: 0.7
- 创新性: 6.5/10

## 优势分析

1. 开头钩子设计出色
2. 对话生动自然
3. 节奏控制得当

## 改进建议

1. 中间部分可适当精简
2. 可增加环境描写增强氛围
3. 结尾悬念可进一步加强
```

```yaml
  - 结尾有悬念: 是/否
  - 可读性评分 ≥ 7: 是/否
通过标准: 5/5通过
```

## 六、最佳实践

### 技能设计原则

1. **单一职责**：每个技能只做一件事
2. **明确接口**：输入输出清晰定义
3. **可测试性**：每个技能都有评测用例
4. **可复用性**：设计时考虑复用场景
5. **可扩展性**：支持功能扩展

### 智能体协作原则

1. **松耦合**：智能体间依赖最小化
2. **高内聚**：相关功能集中管理
3. **容错设计**：错误不影响整体流程
4. **状态可追溯**：所有操作有记录
5. **性能可监控**：关键指标可测量

### 项目管理原则

1. **迭代开发**：小步快跑，持续交付
2. **数据驱动**：决策基于数据分析
3. **用户中心**：始终考虑最终读者
4. **质量优先**：不牺牲质量换速度
5. **持续改进**：每个项目都是学习机会

## 八、并行分卷创作模式

### 适用场景

- 多卷长篇小说（100万字以上）
- 分卷相对独立的作品
- 需要快速完成的作品

### 效率对比

| 模式          | 100万字小说 | 300万字小说 | 效率提升 |
| ------------- | ----------- | ----------- | -------- |
| 串行模式      | 约 5-7 天   | 约 15-21 天 | -        |
| 并行模式(3卷) | 约 2-3 天   | 约 6-9 天   | **60%**  |

### 分卷策略

| 策略      | 说明         | 适用场景       |
| --------- | ------------ | -------------- |
| equal     | 等字数拆分   | 篇幅固定的作品 |
| plot      | 剧情节点拆分 | 情节驱动的作品 |
| character | 人物成长拆分 | 成长型作品     |

### 执行命令

```bash
# 1. 拆分大纲为3卷
/nf-novel-flow split --outline main.md --volumes 3 --strategy plot

# 2. 并行创作3卷
/nf-novel-flow parallel-build --volumes 1,2,3 --max-workers 3

# 3. 合并为完整小说
/nf-novel-flow merge --volumes vol1.md,vol2.md,vol3.md --output novel.md

# 4. 一键执行完整并行流程
/nf-novel-flow parallel-run --name "小说名" --volumes 3 --strategy plot
```

## 九、新手教程

### 快速开始

#### 1. 一键创作（推荐新手）

```bash
/nf auto --name 我的小说 --genre 玄幻 --volumes 3 --chapters 30
```

**参数说明**：
| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `--name` | 是 | - | 小说名称 |
| `--genre` | 是 | - | 题材类型 |
| `--volumes` | 否 | 3 | 分卷数量 |
| `--chapters` | 否 | 30 | 每卷章节数 |
| `--platform` | 否 | - | 发布平台 |

#### 2. 分步创作（推荐进阶用户）

**Step 1: 创意探索**

```bash
/nf idea --genre 玄幻 --count 5
```

输出5个创意方案供选择。

**Step 2: 项目立项**

```bash
/nf init --name 逆天剑尊 --genre 玄幻 --volumes 3 --chapters 30 --target 1000000
```

创建项目目录结构（指定篇幅参数）。

**Step 3: 生成大纲**

```bash
/nf outline --volumes 3 --chapters 30
```

生成完整大纲体系。

**Step 4: 创作正文**

```bash
/nf write --mode parallel
```

并行创作多卷正文。

**Step 5: 质量优化**

```bash
/nf optimize --type full
```

全量优化章节质量。

**Step 6: 发布平台**

```bash
/nf publish --platform qidian
```

适配并发布到目标平台。
